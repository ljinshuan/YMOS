# 总入口暗号

> YMOS V3.1 四层架构：skills（能力层）/ references（知识层）/ data（数据层）/ cli（工具层）
> 所有工作流通过自然语言暗号触发，Agent 读取对应 Skill/SOP 按步骤执行。

---

## 智能路由（意图分类层）

> 用户输入先经过智能路由判断，再进入具体能力。

```
用户输入
  ├── 命中下方暗号？ ──是──→ 直达对应 Skill（最短路径）
  └── 否 ──→ 意图分类器自动识别 → 路由到 Skill
```

**意图分类器支持 8 种意图：** 市场概览 / 投资雷达 / 个股调研 / 买入建仓 / 持有评估 / 卖出减仓 / 组合管理 / 系统操作

**复合意图：** "研究NIO然后看要不要加仓" 会被自动拆分为 `[{调研, NIO}, {买入评估, NIO}]`，按顺序执行。

- 触发词（下方暗号表）优先级最高，命中即直达
- 自然语言走意图分类，无需记忆暗号
- 详见 `skills/ymos-core/prompts/intent-classifier.md`

## Step 0：开始使用

> 第一次使用？把下面这段话复制给你的 AI Agent：

```
请读取 [你的YMOS根目录]/总入口暗号.md ，然后对我说「开始使用」
```

> 例如：`请读取 /Users/xxx/Documents/Obsidian Vault/YMOS/总入口暗号.md`
>
> Agent 读取本文件后，说「开始使用」即可进入入职引导（填写投资偏好 → 添加持仓 → 添加关注 → 跑通闭环）。
> 已经初始化过的用户，直接用下方暗号表操作即可。

## 🔄 链路全景

```
市场洞察 & 投资雷达
  跑一下市场洞察 → data/reports/market-insight/
  跑一下投资雷达 → data/reports/radar/（桥接报告：趋势+价格+下一步建议）
                     ↓
策略分析 & 调研
  跑一下策略分析 ← 读取雷达建议 → data/reports/strategy/（个股报告 + 日志）
  我想买/卖/加仓/持有怎么看 → data/reports/strategy/ → 挪入个股文件夹
  调研一下 → P1+P4+P2+P9 → 个股基础知识库
                     ↓
持仓与关注（管状态）
  关注/建仓/移除/清仓 → 状态机更新 + 个股文件夹管理
  收口一下 → 一致性校验 + 刷新持仓备忘录视图（人读层看板）
  个股文件夹 = 后续分析的完整上下文源
```

**用户日常只看四份报告**：
1. 市场洞察 → 市场发生了什么
2. 投资雷达 → 和我有什么关系 + 下一步做什么
3. 策略分析日志 → 分析了什么 + 待我确认什么
4. 持仓备忘录视图 → 1 分钟扫盘执行看板
