# Watchlist Update Workflow
# Research模块 · 日常运营工作流

> **定位**：这不是一个投资分析提示词，而是一个工作流编排文档。
> 它描述的是如何把 Watchlist 状态机、P4（雷达）、联网搜索、分级解读串联起来，
> 完成 Watchlist 的日常定期更新（日报/周报/月报）。

---

## 触发方式

| 用户说 | 解析 |
|:---|:---|
| "更新过去一周watchlist" | scope=watchlist, period=1w |
| "更新NVDA过去一个月" | scope=NVDA, period=1m |
| "日报" | scope=watchlist, period=1d |
| "周报" | scope=watchlist, period=1w |
| "更新TSLA和NVDA过去两周" | scope=[TSLA, NVDA], period=2w |

**参数：**
- `scope`：单个 ticker / 多个 ticker / "watchlist"（自动读取 `持仓与关注/Watchlist_状态机.md`）
- `period`：1d / 1w / 2w / 1m，默认 1w

---

## 执行流程

```
Step 1：确定标的清单
  - scope=watchlist → 读取 `持仓与关注/Watchlist_状态机.md` 的 Ticker 列
  - scope=ticker(s) → 直接使用指定标的

Step 2：对每个标的，加载分析上下文
  优先读取：
  - `持仓与关注/动态Watchlist/{名称_TICKER}/个股基础知识库.md`
  - `持仓与关注/动态Watchlist/{名称_TICKER}/买入卖出备忘录.md`（若存在）
  - `持仓与关注/当前关注方向与投资偏好.md`
  并结合 P4 雷达框架（skills/ymos-research/prompts/p4-radar.md）提取：
  - 北极星指标及预警阈值
  - 叙事框架（当前叙事 + 强化/弱化信号清单）
  - 竞品清单及解读模板
  - 产业链上下游关注事项
  - Thesis Killer清单
  - 当前策略类型（PVE / PVP，若无则标记待确认）
  ⚠️ 如无基础知识库 → 标注警告，并建议先运行 P1/P4 建档

Step 3：搜索period内的增量事件
  按以下顺序搜索（以P4雷达关注事项为搜索线索）：
  1. 公司官方披露（财报、8-K、月度运营数据）
  2. 管理层动作（insider trading、高管变动）
  3. 公司产品/战略事件
  4. 竞品重大事件（财报、产品发布、定价变动）
  5. 产业链信号（上游价格、下游需求数据）
  6. 宏观/监管事件（利率、政策、地缘政治）

Step 4：事件分级与解读
  对每个事件：
  a) 判断Tier级别（1/2/3，参照P4雷达定义）
  b) 映射到P4雷达的哪个维度
  c) 写出简要解读：事件 → 影响传导 → 对投资逻辑的强化/弱化/中性
  d) 双轨视角：
     - PVE视角：是否影响长期竞争力/护城河？
     - PVP视角：催化剂是否兑现/失效？情绪方向？
  e) 是否触发北极星指标预警
  f) 是否属于Thesis Killer
  → Tier 1重大事件 → 建议进入 P3 做深度事件解读

Step 5：生成输出
  A) 个股更新报告（详细版）
  B) 追踪日志追加（Tier 1摘要置顶，不覆盖历史）
  C) scope=watchlist时，额外生成跨标的汇总报告
```

---

## 与其他模块的衔接

| 发现信号类型 | 建议后续动作 | 目标提示词 |
|:---|:---|:---|
| Tier 1事件（财报/重大政策/Thesis Killer） | 触发深度事件分析 | P3（事件动态） |
| 叙事状态变化（强化/弱化/裂缝） | 更新叙事验证状态 | P2（阶段定性） |
| 北极星指标异常 | 触发事件洞察 | P15（深度洞察） |
| 财报日到来 | 触发财报深度解读 | P16（财报进化） |
| 持仓逻辑出现根本变化 | 触发持仓评估 | P6（持仓与卖出） |

## 输出格式

### 个股更新报告（写入标的文件夹 `个股基础知识库.md` 的增量段）

```markdown
# [ticker] 更新报告 · [date]
> 覆盖周期：[period] | 策略类型：[PVE/PVP]

## Tier 1 信号（需立即关注）
- [事件] → [解读] → [建议动作]

## Tier 2 信号（本周纳入分析）
- [事件] → [解读]

## 叙事状态更新
当前叙事：[一句话]
状态：[强化 / 待验证 / 出现裂缝 / 被证伪]
下一个Checkpoint：[事件 + 时间]

## 北极星指标
[指标]：[当前值] [是否触发预警]
```

### Watchlist 汇总报告（scope=watchlist 时额外输出，写入 `Eyes/投资雷达/YYYY-MM/`）

```markdown
# Watchlist 周报 · [date]

## 需要立即处理
- [ticker]：[Tier 1信号摘要]

## 叙事状态总览
| 标的 | 叙事状态 | 下一个Checkpoint |
|------|---------|----------------|
| [ticker] | 强化/待验证/裂缝/证伪 | [事件+时间] |

## 本周无重大变化
- [ticker列表]
```

## 降级策略

| 情况 | 处理方式 |
|:---|:---|
| 标的无P4雷达 | 完成更新但质量受限，追加"建议先运行 P4 Radar" |
| 联网搜索失败 | 基于知识库完成，标注"联网失败，数据可能非最新" |
| 标的无P1档案 | 完成基础信息更新，标注"建议先运行 P1 Genesis 建立档案" |
