# P11: 投资交易复盘 (Autopsy)



## 路径上下文（YMOS）
- 根目录：`YMOS/`
- 主数据目录：`持仓与关注/`
- 前置读取：`个股基础知识库.md` + `买入卖出备忘录.md` + `当前关注方向与投资偏好.md`
- 写回：标的 `个股基础知识库.md`（追加 P11 复盘区块）+ `当前关注方向与投资偏好.md`（心法追加，需用户确认）

## 适用场景

卖出/清仓后的强制复盘（Route D 必经步骤）。区分"运气"与"实力"，将交易经验回流到系统中。

## 触发规则

| 场景 | 是否触发完整复盘 |
|:---|:---|
| 完全清仓（标的从持仓移入 Watchlist） | **强制执行**四维度完整复盘 |
| 部分减仓（标的仍在持仓中） | 简版提示（买入逻辑是否仍然成立） |

## 提示词模板

```
# Role

请对刚刚结束的一笔交易 **[{{ticker}}]** 进行"尸检"。目标不是安慰我，而是找出系统漏洞，把经验回流到投资体系里。

# Context (User Input)
1. **Ticker:** {{ticker}}
2. **Result:** {{result}}
3. **Original Thesis:** {{original_thesis}}
4. **Exit Reason:** {{exit_reason}}
5. **P1 Genesis (建仓时):** {{p1_at_entry}}
6. **P4 Radar (建仓时):** {{p4_at_entry}}
7. **P5 FOMO Audit (买入时):** {{p5_result}}
8. **P6 Profit Keeper (持有期间):** {{p6_results}}

# Structured Audit Protocol（四维度复盘）

## 维度一：买入逻辑回溯
* 原始买入理由是什么？（从备忘录中提取）
* 实际兑现情况：哪些逻辑被验证了？哪些被证伪了？
* 买入时的核心假设是什么？现实是否符合假设？
* **结论**：[逻辑兑现 / 逻辑部分兑现 / 逻辑证伪] + 一句话说明

## 维度二：P1/P4 判断偏差
* 建仓时 P1 基石档案的判断：哪些准确？哪些偏差大？
* 建仓时 P4 关注点雷达：遗漏了什么？过度关注了什么？
* 有没有"建档时没注意到但后来成为关键因素"的东西？
* **结论**：列出 [准确项] / [偏差项] / [遗漏项]

## 维度三：P5/P6 阈值校准
* P5 FOMO Killer 买入审计：当时通过是否合理？是否过于宽松？
* P6 Profit Keeper 持有期间：止损/止盈触发条件是否合适？
* 如果重新设定阈值，你会怎么调？
* **结论**：[阈值合理 / 建议调整] + 具体调整建议

## 维度四：经验教训提炼
* 这笔交易最大的收获是什么？（用 2-3 句话）
* 应该形成什么纪律/规则？（可执行的，不是空话）
* 这个教训应该写入投资偏好的哪个位置？
```

## 输入参数

| 参数 | 说明 | 必填 |
|-----|------|-----|
| `ticker` | 股票代码 | 是 |
| `result` | 交易结果（盈利XX%/亏损XX%） | 是 |
| `original_thesis` | 当初买入的理由（从买入卖出备忘录提取） | 是 |
| `exit_reason` | 实际卖出的理由 | 是 |
| `p1_at_entry` | 建仓时的 P1 基石档案 | 是（从知识库读取） |
| `p4_at_entry` | 建仓时的 P4 关注点 | 是（从知识库读取） |
| `p5_result` | 买入时的 P5 审计结果 | 否（从备忘录读取，无则标注） |
| `p6_results` | 持有期间的 P6 评估记录 | 否（从备忘录读取，无则标注） |

## 复盘评分标准

| 评分维度 | 权重 | 说明 |
|---------|-----|-----|
| 知行合一 | 40% | 买卖理由是否匹配 |
| 纪律执行 | 30% | 是否按计划止损/止盈 |
| 逻辑验证 | 30% | 投资逻辑是否得到验证 |

## 写回规则（强制）

复盘完成后**必须**执行以下写回：

1. **个股知识库**：在 `个股基础知识库.md` 末尾追加 `## P11 交易复盘` 区块
   ```markdown
   ## P11 交易复盘
   > 复盘时间：YYYY-MM-DD

   - **交易周期**：YYYY-MM-DD 至 YYYY-MM-DD（XX 天）
   - **收益率**：XX%
   - **交易评分**：X/10
   - **买入逻辑兑现**：[兑现 / 部分兑现 / 证伪]
   - **P1/P4 偏差**：[准确项 / 偏差项 / 遗漏项]
   - **核心教训**：[一句话]
   ```

2. **投资偏好**（需用户确认后写入）：
   - 如果复盘发现阈值需要调整 → 追加到「核心心法」→「历史教训」表格
   - 如果复盘发现新的纪律规则 → 追加到「当前禁忌」
   - ⚠️ 投资偏好写入必须经用户确认

## 常见系统漏洞

| 漏洞类型 | 表现 | 改进方向 |
|---------|-----|---------|
| 调研不深 | 买入后才发现关键风险 | 强化 P1 调研流程 |
| FOMO 入场 | 追高买入 | 强化 P5 审计 |
| 不止损 | SOP 2 变 SOP 1 | 强化 P12 纪律检查 |
| 过早止盈 | PVE 主升浪中被洗出 | 强化 P6 持仓评估 |
| 阈值过松 | P5/P6 频繁通过不该通过的交易 | 复盘后校准阈值参数 |
| 忽视关联 | 买了"不同"股票实为同一笔交易 | P17 集中度检查 |
