# 入职引导 SOP

> 暗号：`开始使用` / `初始化系统`
> 路径基准：`YMOS/`

---

## 一句话定位

入职引导是系统的「冷启动」流程：**了解用户关注什么、持有什么、怎么想的**，初始化整个系统。

- 只在首次使用或系统状态为空时触发
- 完成后用户可以直接进入日常流程（`跑一下市场洞察`）
- 核心使命：**引导用户写好 `当前关注方向与投资偏好.md`**

---

## 触发方式

### 方式一：自动检测（每次新会话）

Agent 在会话开始时执行以下检测：

1. `data/当前关注方向与投资偏好.md` — 是否有实质内容（不只是模板）
2. `data/持仓_状态机.md` — 是否有持仓行
3. `data/Watchlist_状态机.md` — 是否有关注行

**判定规则**：
- 三者全空 → 触发 **完整入职引导**
- 投资偏好为空 → 触发 **偏好补全引导**（最高优先）
- 仅缺持仓或 Watchlist → 提示补全，不强制

### 方式二：用户主动触发

| 暗号 | 操作 |
|:---|:---|
| `开始使用` | 完整入职引导 |
| `初始化系统` | 完整入职引导 |
| `补全信息` | 补全引导（只补缺失部分） |

---

## 完整入职引导步骤

### Step 1：投资偏好深度访谈（最重要）

> 这一步决定了后续所有分析的质量。必须引导用户充分表达。

**引导语**：
```
欢迎使用 YMOS！我是你的投资研究助手。

在开始之前，我需要深入了解你的投资风格和当前想法。
这些信息会写入「当前关注方向与投资偏好.md」—— 它是整个系统的灵魂文件，
投资雷达靠它判断信号相关性，策略分析靠它校准建议，纪律审查靠它拦住冲动。

填得越详细，系统越懂你。我会分几个维度来问你：
```

**结构化访谈 9 维度**（对应文件的 9 个 section）：

**1. 投资者画像**
```
你怎么描述自己的风险承受度？（低/中低/中/中高/高）
你的投资周期偏好？（短线几天 / 波段几周 / 中线几月 / 长线跨年）
资金属性？（闲钱 / 有时间压力 / 有杠杆）
你觉得自己投资上最大的弱点是什么？（追涨/恋战/FOMO/过度交易/其他）
```

**2. 仓位配置框架**
```
你目前大概多少比例的资金在股市里？（整体仓位占比）
你理想的仓位分配是怎样的？（比如：长线X% / 短线Y% / 现金Z%）
你能接受单只股票最大占多少？（单标的上限）
```

**2.5 仓位管理参数**（可选，使用默认值也可以）
```
系统有一个仓位计算器（P17），会在你买入/加仓时自动计算建议仓位。
它需要几个参数，你可以自定义，也可以使用默认值：

- 单笔建仓上限：单次买入占总资金的最大比例（默认 10%）
- 单标的集中度上限：单只股票占总资金的最大比例（默认 20%）
- 单板块集中度上限：同一板块占总持仓的最大比例（默认 30%）
- 分批建仓规则：建仓分几批、每批触发条件（默认分 2 批，第二批在盈利 5% 后）
- 现金储备下限：始终保留的现金比例（默认 10%）

你想用默认值，还是有自己的偏好？之后随时可以改。
```

**3. 策略立场**
```
你当前对市场是偏进攻还是偏防守？
你怎么区分长线持仓和短线博弈？有什么进入和退出标准？
```

**4. 关注方向与板块偏好**
```
你目前最关注哪些投资方向或板块？（2-4 个）
每个方向的核心逻辑是什么？（一两句话）
有哪些市场信号是你觉得跟你有关的？（比如：英伟达财报→映射XX，BTC站稳→影响XX）
宏观层面，你最关心什么？（利率/地缘/汇率等）
```

**5. Watchlist 偏好**
```
你把一只股票加入关注清单的标准是什么？
什么情况下你会把它移出清单？
```

**6. 执行纪律**
```
你买入前会做哪些检查？
卖出呢？
有没有什么你给自己定的硬性纪律？
```

**7. 禁忌**
```
有什么你绝对不碰的？（行业/操作方式/市场条件）
```

**8. 核心心法**
```
你投资生涯中最深刻的教训是什么？（用真金白银换来的经验）
有没有你的投资信念或座右铭？
```

**9. 热门题材与反向信号**（可选，帮助 AI 更精准匹配市场信号）
```
除了上面的板块方向，你还在跟踪哪些热门题材？（未形成具体标的但在关注的大方向，比如商业航天、加密货币等）
每个题材的关键词是什么？（AI 看到这些词会帮你高亮）
有什么类型的新闻/信号你看到就想远离的？（反向信号，比如"纯概念炒作"）
```

**写入**：根据用户回答，填充 `data/当前关注方向与投资偏好.md` 的各 section。

> ⚠️ **写入前必须将整理好的完整内容展示给用户，获得明确确认后再写入文件。** 这是用户的投资灵魂文件，Agent 不得静默写入。

> 如果用户某些维度回答得简略，Agent 应追问一两句引导展开。这个文件不怕长，怕薄。

### Step 2：当前持仓

```
你目前持有哪些标的？
请告诉我 Ticker 或名称，逗号分隔即可。
例如：NIO, 腾讯, NVDA, BTC

如果当前空仓，直接说「空仓」即可。
```

对每个持仓标的：
1. 确认 Ticker、名称、所属市场
2. 调用 `data/SOP_标的管理.md` 的「建仓」流程
3. 询问是否立即跑初始调研（P1+P4+P2）
   - 标的 ≤3 个：建议同步跑，直接说 `调研一下 [TICKER]`
   - 标的 >3 个：先做基础建档，深度调研后续按需 `调研一下 [TICKER]`

### Step 3：关注清单

```
除了已持有的，你还在观察哪些标的？（尚未买入但在关注的）
同样 Ticker 或名称，逗号分隔。没有的话说「暂无」。
```

对每个关注标的：
- 调用 `data/SOP_标的管理.md` 的「关注」流程
- 同理，询问是否立即跑初始调研（说 `调研一下 [TICKER]` 即可触发）

### Step 4：确认与引导下一步

```
✅ YMOS 初始化完成！

📊 你的投资工作台已就绪：
- 投资偏好：已配置（9 维度）
- 持仓：X 个标的
- 关注：Y 个标的

💡 下一步你可以：
1. 说「跑一下市场洞察」— 看看今天市场发生了什么
2. 说「跑一下投资雷达」— 看看持仓和关注的价格动态
3. 说「调研一下 [TICKER]」— 深入研究某个标的
```

## 补全引导模式

当系统检测到部分数据已填时，只补缺失的部分：

| 缺失项 | 补全动作 |
|:---|:---|
| `当前关注方向与投资偏好.md` 为空/过于简略 | **最高优先**，执行 Step 1 |
| 持仓状态机为空 | 执行 Step 2 |
| Watchlist 为空 | 执行 Step 3 |

补全模式的消息更简洁：
```
检测到你的 [缺失项] 尚未配置。我来帮你快速补全。
```

### 偏好文件「简略」判定标准

Agent 检查 `当前关注方向与投资偏好.md` 时，以下情况视为"过于简略，需要补全"：
- 缺少「投资者画像」section
- 缺少「仓位配置框架」section
- 缺少「核心心法」section
- 「板块关联逻辑」表格为空

提示用户补全时的引导语：
```
你的「当前关注方向与投资偏好」文件还比较薄。
这个文件是系统的灵魂——填得越详细，投资雷达越精准，策略建议越对味。
我来帮你逐项补充几个关键维度，大约需要 5-10 分钟。
```

---

## 边界

- 入职引导**不做市场分析**（那是市场洞察的事）
- 入职引导**不做策略推荐**（那是策略分析的事）
- 只负责：了解用户 → 初始化状态 → 写入偏好文件 → 引导下一步
- 持仓/关注的建档复用 `SOP_标的管理.md`，不重复实现
