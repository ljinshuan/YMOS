# ymos-onboarding：入职引导

## 触发
- `开始使用` — 完整入职引导
- `初始化系统` — 完整入职引导
- `补全信息` — 仅补全缺失部分

## 前置条件
- 无（这是系统的入口 skill）

自动检测逻辑（每次新会话）：
1. `data/当前关注方向与投资偏好.md` 是否有实质内容
2. `data/持仓_状态机.md` 是否有持仓行
3. `data/Watchlist_状态机.md` 是否有关注行

三者全空 → 完整引导；投资偏好为空 → 优先补全；仅缺持仓/Watchlist → 提示补全。

## 执行步骤
> 详细步骤见 sop.md

1. **投资偏好深度访谈**（最核心）— 结构化 9 维度访谈，引导用户填写 `当前关注方向与投资偏好.md`
   - 投资者画像 / 仓位配置框架 / 仓位管理参数 / 策略立场 / 关注方向 / Watchlist 偏好 / 执行纪律 / 禁忌 / 核心心法
   - 写入前必须将完整内容展示给用户确认
2. **当前持仓录入** — 确认 ticker、市场，调用 `ymos-target-mgmt` 的建仓流程
3. **关注清单录入** — 调用 `ymos-target-mgmt` 的关注流程
4. **确认与引导下一步** — 提示用户可以跑市场洞察、投资雷达或调研

## 产出物
- `data/当前关注方向与投资偏好.md`（用户确认后写入）
- `data/持仓_状态机.md`（新增持仓行）
- `data/Watchlist_状态机.md`（新增关注行）
- 各标的文件夹（`data/{持仓,动态Watchlist}/名称_TICKER/`）

## 边界
- 不做市场分析、不做策略推荐
- 持仓/关注的建档复用 `ymos-target-mgmt`，不重复实现
- 偏好文件写入前必须获得用户明确确认（Human-in-the-Loop）
