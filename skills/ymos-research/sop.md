# 🔬 初始调研 SOP（可复用子模块）

> 暗号：`调研一下 [ticker]`（也可被其他 SOP 按需调用）
> 模块：ymos-research（初始调研子模块）

---

## 一句话定位

初始调研是**可复用子模块**，为任意标的生成 P1 基石档案 + P4 关注点雷达 + P2 阶段判断。

- 可由用户直接触发（`调研一下 XX`），也可被策略分析 / 标的管理按需调用
- 设计原则：**AI 先做到最大程度的自主分析，只把真正做不了的交给用户**

---

## 🔑 触发暗号

| 暗号 | 操作 |
|:---|:---|
| `调研一下 [ticker/名称]` | 深度调研（P1 → P4 → P2 + 可选 P9） |

---

## 📥 调用接口

| 调用方 | 触发场景 | 传入参数 |
|:---|:---|:---|
| 用户直接 | `调研一下 XX` | ticker + 名称 |
| 标的管理_SOP | `关注 XX` / `建仓 XX` 时触发 | ticker + 名称 + 位置 |
| 策略分析_SOP | 策略路由发现缺 P1/P4 | ticker + 位置 |
| 定时任务 | 投资雷达建议调研的标的 | ticker + 名称 + 位置 |

## ⚙️ 执行步骤

### Step 1：确认标的信息

- 解析 ticker 和中文名称
- 确认位置：`data/stocks/holdings/名称_TICKER/` 或 `data/stocks/watchlist/名称_TICKER/`
- 确认目录存在（不存在则创建）

### Step 2：检查个股基础知识库

- 读取 `data/stocks/{holdings,watchlist}/名称_TICKER/个股基础知识库.md`
- 若不存在：从模板创建最小骨架

```markdown
# 个股基础知识库 - [名称]（[TICKER]）

## P1 基石档案
（待生成）

## P4 重点关注点
（待生成）

## P2 阶段判断
（待生成）

## P9 估值分析
（用户调研时生成）
```

### Step 3：执行 P1 Genesis 基石建档

**读取提示词**：`prompts/p1-genesis.md`

AI 自主完成（利用搜索能力补足）：
- 治理结构：CEO 背景、核心团队、股东结构
- 商业壁垒：商业模式、护城河分析
- 财务健康：营收/利润/现金流趋势、北极星指标
- 周期维度：行业阶段、宏观敏感度
- 机会分类：PVE/PVP 初判、中美映射关系

**写入位置**：`个股基础知识库.md` 的 `## P1 基石档案` 区块

> AI 搜索补足规则：
> - 优先搜索：最新财报、SEC filings、行业报告、竞品动态
> - 标注信息来源和时效性
> - 搜索不到的关键信息 → 标记为「待人工补充」

### Step 4：执行 P4 Radar 关注点雷达

**读取提示词**：`prompts/p4-radar.md`

以 P1 产出为 context 输入，生成五维度关注清单：
1. **本体核心（Core）**
2. **叙事验证（Narrative）**
3. **直接竞品（Mirror）**
4. **产业链（Chain）**
5. **宏观与行业（Macro）**

**写入位置**：
1. `个股基础知识库.md` 的 `## P4 重点关注点` 区块
2. **同步更新状态机**：P4 摘要写入对应状态机的 `P4重点关注点` 列

### Step 5：执行 P2 Phase Check 阶段判断

**读取提示词**：`skills/ymos-core/prompts/p2-phase-check.md`

AI 自主完成（基于 P1+P4 + 搜索最新数据）：
- PVE/PVP 特征扫描
- 玩家识别
- 叙事验证状态
- 三情景预演（Bull/Base/Bear）

**写入位置**：`个股基础知识库.md` 的 `## P2 阶段判断` 区块

### Step 5.5：执行 P9 估值分析（仅 `调研一下` 直接调用时）

**读取提示词**：`skills/ymos-core/prompts/p9-valuation.md`

执行反向 DCF 估值，产出写入 `## P9 估值分析` 区块。

> 被标的管理/策略分析调用时跳过此步（P9 在策略路由中按需触发）。

### Step 6：输出确认

```
✅ [名称]（[TICKER]）初始调研完成
- P1 基石档案已生成（AI 搜索补足）
- P4 关注点雷达已生成（五维度）
- P2 阶段判断已完成：[PVE/PVP/混合]
- 状态机 P4 摘要已同步
[- P9 估值分析已完成（仅调研模式）]

📋 待人工补充（如有）：
- [具体待办项]
```

### 状态机写回最小闭环（强制）

每次更新状态机时必须：
1. 更新顶部 `更新时间`
2. 更新对应标的所在行
3. 在 `今日变更日志` 中追加一条摘要

## 🤖 AI 自主程度

| 模块 | AI 自主度 | 说明 |
|:---|:---|:---|
| P1 Genesis | ~60% | AI 搜索补足公开信息；深度研报需人工 |
| P4 Radar | ~80% | 基于 P1 几乎全自动 |
| P2 Phase Check | ~90% | 搜索价格+资金流数据后自主判断 |

---

## ⚠️ 边界

- 初始调研**只做 P1+P4+P2**（直接调用时加 P9）
- 不做策略制定（那是策略分析 SOP 的事）
- 不修改状态机的身份/状态列（那是标的管理 SOP 的事），只更新 P4 关注点列
