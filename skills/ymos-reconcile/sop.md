# SOP：持仓收口

> 每日 pipeline 最后一步。策略分析已经写回了状态机和备忘录，本 SOP 做**视图生成 + 一致性校验**。

---

## 触发暗号

- `收口一下`
- `刷新持仓视图`
- `跑一下持仓收口`

## 前置条件

当日至少有以下产出之一：
- 市场洞察报告（`Eyes/市场洞察/YYYY-MM/`）
- 投资雷达报告（`Eyes/投资雷达/YYYY-MM/`）
- 策略分析报告（`Brain/策略分析/YYYY-MM/`）

如果当日没有任何新增产出，仍可运行（仅刷新视图），但需告知用户"今日无新增分析"。

---

## Step 1：读取当前状态

读取以下文件：

| 文件 | 用途 |
|:---|:---|
| `持仓与关注/持仓_状态机.md` | 当前持仓状态 |
| `持仓与关注/Watchlist_状态机.md` | 当前关注状态 |
| `Brain/策略分析/YYYY-MM/` | 当日策略分析报告（如有） |
| `Eyes/投资雷达/YYYY-MM/` | 当日投资雷达报告（如有） |
| 各持仓标的的 `买入卖出备忘录.md` | 最新操作记录 |

---

## Step 2：一致性校验

检查策略分析的写回是否已正确反映在状态机中：

1. **时间戳检查**：状态机顶部 `更新时间` 是否为当日
2. **标的行检查**：策略分析涉及的标的，其状态机行是否已更新（阶段/价格/监控位等）
3. **变更日志检查**：`今日变更日志` 是否已追加策略分析的摘要

**如有遗漏**：
- 补写状态机（强制三步：更新时间 + 标的行 + 变更日志）
- 在视图的"缺口清单"中记录发现的遗漏

---

## Step 3：生成持仓备忘录视图

覆盖写入：`持仓与关注/持仓备忘录_视图.md`

### 视图结构

```markdown
# 持仓备忘录（执行看板）

> 用途：给你"1 分钟扫盘"的执行看板（人读层）
> 真相源：`持仓_状态机.md`（机读层）
> 更新时间：YYYY-MM-DD HH:MM

---

## 今日驾驶舱

- 持仓数：X 只
- 组合状态：[防守/进攻/平衡]
- 执行灯号：[绿灯=无待办 / 黄灯=有待确认 / 红灯=有紧急动作]

---

## 持仓卡片

### [标的名称] ([TICKER])
- **仓位身份**：核心仓 / 卫星仓 / 观察仓
- **当前阶段**：P2 阶段判断结论
- **三要点**：① ... ② ... ③ ...
- **失效条件**：[什么情况下逻辑不再成立]
- **最新动作**：[来自买入卖出备忘录的最新记录]

（每只持仓一张卡片）

---

## 缺口清单

### 数据缺口
- [ ] [缺失的数据/分析]

### 结构缺口
- [ ] [缺失的个股知识库/备忘录等]

---

## 今日动作队列

| 优先级 | 动作 | 标的 | 来源 |
|:---|:---|:---|:---|
| P0 | [紧急] | ... | 策略分析/投资雷达 |
| P1 | [重要] | ... | ... |
| P2 | [常规] | ... | ... |

---

> 本文件是执行看板，不是分析报告。分析详情见各标的的策略分析报告和个股知识库。
```

### 写入规则

- **同日覆盖**：每次运行都覆盖整个文件
- **真相源**：所有数据从状态机和备忘录读取，视图不产生新信息
- **缺口清单**：如果某只持仓缺少个股基础知识库、买入卖出备忘录、或 P2 阶段判断，在这里标记

---

## Step 4：生成 Dashboard（可视化看板）

> 默认执行（token 消耗可忽略，因为此时 4 份报告已在上下文中）。
> 用户说 `不需要 Dashboard` 或 `跳过看板` 时可跳过。

### 数据来源

你在 Step 1-3 已经读取了所有需要的文件。无需重新读取。

### 输出

生成一个**自包含 HTML 文件**（CSS 内嵌，无外部依赖），写入：

```
持仓与关注/dashboard/YYYY-MM/dashboard_YYYY-MM-DD.html
```

### 布局要求

生成的 HTML 须包含以下区块，从上到下排列：

**1. 顶栏**
- 左侧：`YMOS │ 投资操作系统`
- 右侧：当日日期

**2. 状态概览（卡片）**
- 左侧：状态灯（🟢 无待办 / 🟡 有待确认 / 🔴 有紧急动作）+ 状态文字
- 右侧：持仓数量

**3. 每日工作流（卡片）**
- 4 个步骤横向排列：市场洞察 → 投资雷达 → 策略分析 → 持仓收口
- 已完成的步骤：绿色背景 + ✅ + "今天已完成"
- 未完成的步骤：灰色背景 + 序号 + "点击复制指令"
- 底部：`已完成 N/4`

**4. 战略简报（卡片，来自市场洞察）**
- 市场体温标签 + 一句话摘要
- 战略信号仪表盘表格（信号类型 | 核心主题 | 研判结论）

**5. 晨会速览（卡片，来自备忘录视图）**
- 左列："今天必须做的事"
- 右列："今天不要做的事"

**6. 需要你做的事（卡片）**
- 来源：备忘录视图的动作队列 + 投资雷达的策略/调研建议
- 每条一行：优先级标签（P0 红色 / P1 橙色 / P2 灰色）+ 动作描述 + 右侧标的/范围

**7. 持仓一览（卡片）**
- 每只持仓一行：
  - 左侧：标的名称 + Ticker + 阶段（PVE/PVP + 具体阶段）
  - 右侧：最新动态/状态摘要

**8. 页脚**
- `YMOS Dashboard · 数据截至 YYYY-MM-DD HH:MM`

### 样式要求

- **页面宽度**：内容区最大宽度 `1200px`，居中，左右 padding `24px`（充分利用屏幕空间）
- **基础字号**：body `16px`，卡片正文 `15px`，表格内容 `14px`，标题按层级递增
- **行高**：正文 `1.7`，表格 `1.5`（确保中文阅读舒适）
- 背景色：`#f5f5f0`（暖灰）
- 卡片：白色背景、圆角 12px、轻投影、内边距 `24px`
- 字体：系统字体栈（`-apple-system, "PingFang SC", "Microsoft YaHei", sans-serif`）
- 表格：宽度 `100%`，单元格 padding `10px 12px`
- 移动端适配：工作流步骤在窄屏时 2×2 排列
- 中文为主，Ticker/价格保留英文

### 降级规则

- 当日某份报告不存在时，对应区块显示"暂无数据"，不留空
- 所有数据从已读取的文件中提取，不编造

---

运行完成后，向用户确认：

```
✅ 持仓收口完成
- 一致性校验：[通过 / X 处遗漏已补写]
- 持仓备忘录视图已刷新
- Dashboard 已生成：持仓与关注/dashboard/YYYY-MM/dashboard_YYYY-MM-DD.html
- 缺口清单：[X 项数据缺口 / Y 项结构缺口 / 无缺口]
```
