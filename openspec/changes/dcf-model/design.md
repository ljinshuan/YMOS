## Context

YMOS 的 P9 估值分析提供了基本的估值思路，但缺乏量化模型。投资者需要：
1. 可验证的估值计算过程
2. 敏感性分析了解假设的影响
3. 多情景对比评估风险
4. Excel 模型便于修改和共享

Financial Services 的 DCF Model 提供了：
- 5-10 年现金流预测
- WACC 详细计算
- 两种终值计算方法
- 双变量敏感性分析
- 三情景对比

YMOS 需要建立类似的 DCF 建模能力，但保持适配个人投资者的使用场景。

## Goals / Non-Goals

**Goals:**
- 建立完整的 DCF 估值模型
- 提供 WACC 计算功能
- 支持多种终值计算方法
- 提供敏感性分析和情景分析
- 生成专业级 Excel 模型

**Non-Goals:**
- 不提供实时估值更新（手动触发）
- 不实现行业特定的 DCF 变体（如银行、地产）
- 不替代专业金融模型的复杂性（保持易用性）

## Decisions

### 1. 现金流预测期

**决策：** 默认 5 年，可扩展至 10 年

**理由：**
- 5 年平衡了预测可靠性和计算复杂度
- 10 年适用于成熟、可预测的公司
- 与 Financial Services 一致

**预测项目：**
- 收入增长率
- EBITDA 或营业利润
- 折旧与摊销
- 资本支出
- 营运资本变化
- 自由现金流

### 2. WACC 计算

**决策：** 提供标准 WACC 计算公式，支持自定义参数

**公式：**
```
WACC = (E/V × Re) + (D/V × Rd × (1 - Tc))
```

**参数：**
- E/V = 权益占比（基于市值）
- Re = 权益成本（CAPM 模型）
- D/V = 债务占比（基于账面价值或市值）
- Rd = 债务成本（利率）
- Tc = 企业所得税率

**CAPM 模型：**
```
Re = Rf + β × (Rm - Rf)
```

**理由：**
- 标准金融机构使用的方法
- 参数可调整，适应不同市场
- 提供合理的默认值

### 3. 终值计算

**决策：** 支持两种方法，默认使用永续增长法

**方法 1：永续增长法**
```
TV = FCF_n × (1 + g) / (WACC - g)
```

**方法 2：退出倍数法**
```
TV = EBITDA_n × Exit Multiple
```

**理由：**
- 两种方法互补，互为验证
- 永续增长法适合持续经营的公司
- 退出倍数法适合有明确并购预期的公司

### 4. 敏感性分析

**决策：** 提供双变量敏感性表

**默认变量：**
- WACC vs. 永续增长率
- 收入增长率 vs. EBITDA 利润率
- 终值倍数 vs. WACC

**理由：**
- 双变量最常用，足以识别关键风险
- Excel 条件格式直观展示估值范围
- 与 Financial Services 一致

### 5. 情景分析

**决策：** 提供三情景对比（乐观/基准/悲观）

**情景定义：**
- **乐观**：高增长、高利润率、低 WACC
- **基准**：基于历史数据和分析师预期
- **悲观**：低增长、低利润率、高 WACC

**输出：**
- 三情景的估值对比
- 概率加权估值（用户可分配概率）

**理由：**
- 情景分析比单点估计更有价值
- 概率加权提供更现实的期望值
- 符合投资机构的实践

### 6. Excel 模型结构

**决策：** 多 Sheet 结构

**Sheet 1：Assumptions**
- 所有输入参数
- 数据来源注释

**Sheet 2：Free Cash Flow**
- 5-10 年现金流预测
- 关键驱动因素

**Sheet 3：WACC**
- CAPM 计算
- 权益/债务成本分解

**Sheet 4：Terminal Value**
- 永续增长法计算
- 退出倍数法计算

**Sheet 5：Valuation**
- 现值汇总
- 每股价值

**Sheet 6：Sensitivity**
- 双变量敏感性表

**Sheet 7：Scenarios**
- 三情景对比

**理由：**
- 逻辑清晰，便于理解和修改
- 与专业模型结构一致
- 支持 excel-output skill

### 7. 与现有 Skill 集成

**决策：** 创建独立的 `ymos-dcf-model` skill

**集成点：**
- `ymos-research`: P9 估值时询问是否生成 DCF 模型
- `ymos-strategy`: 买入/卖出分析时引用 DCF 估值
- `excel-output`: 使用 Excel 输出生成模型

**理由：**
- 独立 skill 保持可测试性
- 可选调用，不影响现有流程

### 8. 默认参数

**决策：** 提供合理的默认参数，用户可覆盖

**默认值：**
- 无风险利率 (Rf): 3% (10年期国债)
- 市场风险溢价 (Rm - Rf): 6%
- Beta: 行业平均（根据标的公司所属行业）
- 永续增长率 (g): 2.5% (长期通胀率)
- 企业所得税率: 25% (中国标准)

**理由：**
- 提供合理的起点
- 用户可基于市场情况调整
- 减少输入负担

## Risks / Trade-offs

### Risk 1: 预测可靠性

**风险：** 现金流预测高度依赖假设，可能不准确

**缓解：**
- 强调假设的重要性
- 提供敏感性分析展示影响
- 建议使用保守假设
- 鼓励与历史数据对比

### Risk 2: WACC 参数不确定性

**风险：** Beta、市场风险溢价等参数难以准确估计

**缓解：**
- 提供行业平均值作为参考
- 允许用户调整参数
- 提供参数敏感性分析

### Risk 3: 终值占比较高

**风险：** 终值可能占总估值的 50-80%，放大不确定性

**缓解：**
- 使用两种终值方法交叉验证
- 限制永续增长率（不超过长期 GDP 增长率）
- 强调终值的敏感性

### Risk 4: 模型复杂度

**风险：** DCF 模型可能过于复杂，普通投资者难以理解

**缓解：**
- 提供简洁的摘要输出
- 使用直观的可视化
- 提供详细的解释文档

## Open Questions

1. **数据源：** 哪些数据需要从外部获取？
   - 建议：历史财务数据（收入、利润率、资本支出）从数据源获取
   - 建议未来增长率、利润率预期由用户输入或从分析师预期获取

2. **行业特定模型：** 是否需要针对不同行业提供定制模板？
   - 建议：V2 功能，当前使用通用模型

3. **实时更新：** 是否需要根据最新股价更新估值？
   - 建议：手动触发，避免自动更新

4. **模型验证：** 如何验证 DCF 模型的合理性？
   - 建议：与市场估值、可比公司估值对比

5. **历史跟踪：** 是否需要跟踪历史 DCF 预测 vs 实际？
   - 建议：V2 功能，用于模型改进

6. **多货币支持：** 是否需要支持多货币？
   - 建议：当前只支持本地货币（人民币/港币/美元），未来扩展
