## Context

YMOS 的 P8 Macro Filter 是策略分析链条中的宏观压力测试环节，当前采用单向分析：从贴现率敏感度、业务传导机制、避险属性三个维度做高风险/中性/有利评级。这种单视角分析容易受确认偏误影响——投资者倾向于只看到支持自己持仓方向的宏观信号。

FinceptTerminal 用 6 个经济学派 Agent 做多视角辩论。对于 YMOS，不需要 6 个学派，选取 2-3 个最具对立性的视角即可有效打破确认偏误。

## Goals / Non-Goals

**Goals:**
- 在 P8 中引入对立经济学派视角，强制 steelman 反对论点
- 输出"最强反对论点"和"该论点为何被否决/被采纳"
- 保持 P8 输出格式向后兼容（在原有评级基础上增加辩论字段）
- 控制 Token 成本增量在合理范围

**Non-Goals:**
- 不做 6 个学派的全面辩论（太贵且边际收益低）
- 不改变 P8 在策略链条中的位置
- 不引入独立的经济学派 agent

## Decisions

### D1: 选择 3 个对立经济学派

**选择**：凯恩斯主义（需求侧）、奥地利学派（供给侧/自由市场）、重商主义（国家战略）

**理由**：这三个学派对同一宏观事件（如"美联储加息"）的传导路径和投资含义完全不同，形成最强张力。

**备选方案**：
- 2 个（凯恩斯 + 奥地利）→ 可行但缺少地缘/贸易维度
- 6 个全部 → Token 成本过高，边际收益递减

### D2: 辩论模式的触发方式

**选择**：每次 P8 执行都自动触发辩论模式

**理由**：如果让用户选择是否触发，大多数用户会跳过（便利偏好），削弱了打破确认偏误的效果。

**备选方案**：
- 仅在宏观判断"高风险"时触发 → 错过"看似中性但实际有隐藏风险"的场景
- 用户手动触发 → 使用率会很低

### D3: 输出结构设计

**选择**：在原有 P8 三维度评级之后，增加"辩论环节"section，包含：
1. 三个学派的独立判断（各 2-3 句）
2. 最强反对论点（1 个）
3. 该论点的处理（被采纳/被否决 + 理由）
4. 修正后的最终评级（如果辩论改变了原始判断）

**理由**：保持原有输出结构不变，辩论作为增强层叠加，不破坏下游 Route C/E 的消费逻辑。

## Risks / Trade-offs

- **[Token 成本增加 ~50%]** → P8 输出长度增加，但 P8 只在策略分析时触发（非每日流水线），成本可控
- **[分析时间延长]** → 多学派分析需要更多推理时间 → 可接受，策略分析本身不是实时场景
- **[学派观点模板化]** → LLM 可能产出套路化的学派观点 → 通过具体的传导路径要求（必须说明对持仓的具体影响机制）来缓解
