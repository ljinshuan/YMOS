## Context

YMOS 的 P13 Market Scanner 在 5 维信息聚合中的"宏观/政策"维度，对地缘政治事件的分析通常是事件摘要式的（"中东局势紧张，油价上涨"），缺乏结构化的分析框架。这导致地缘政治信号停留在表面，难以转化为对具体持仓的可操作洞察。

FinceptTerminal 用 20 个地缘政治 Agent 覆盖三个分析框架。对于 YMOS，不需要那么多 agent，只需要一个轻量模板，在 P13 检测到地缘政治事件时自动激活。

## Goals / Non-Goals

**Goals:**
- 为 P13 提供结构化的地缘政治分析子模板
- 三步分析框架：地理约束 → 历史模式 → 传导路径
- 条件触发：仅在新闻涉及地缘政治时激活
- 输出必须包含对具体行业/标的的传导路径（可操作性）

**Non-Goals:**
- 不做独立的地缘政治 skill 或 agent
- 不覆盖所有 20 个地区/框架（选取与投资最相关的）
- 不做地缘政治预测（只做传导分析）

## Decisions

### D1: 分析框架设计

**选择**：三步框架，融合 FinceptTerminal 的三个子框架精华：

1. **地理约束**（来自 Prisoners of Geography）：该事件的地理基础是什么？资源分布、海峡通道、邻国关系等硬约束。
2. **历史模式**（来自 World Order）：历史上类似情境如何演变？有哪些先例可参考？（限 1-2 个关键类比）
3. **传导路径**（来自 Grand Chessboard + 投资视角）：→ 影响哪些大宗商品/供应链 → 影响哪些行业板块 → 影响用户持仓中的哪些标的

**理由**：三步框架足够轻量（每个步骤 2-3 句），同时保证从"发生了什么"推导到"对我有什么影响"。

### D2: 触发条件

**选择**：在 P13 的信息聚合阶段，当新闻涉及以下关键词域时自动激活地缘政治模板：
- 军事冲突、制裁、贸易战、关税、领土争端
- 关键地区：中东、台海、南海、乌克兰、朝鲜半岛
- 关键资源：石油、稀土、芯片、粮食

**理由**：通过关键词域做粗筛，由 LLM 做细判断，平衡覆盖率和噪音。

### D3: 输出结构

**选择**：地缘政治分析作为 P13 报告中的一个可选 section：

```
### 地缘政治信号
- **事件**：[一句话概述]
- **地理约束**：[硬约束分析]
- **历史回声**：[1-2 个关键历史类比]
- **传导路径**：[事件] → [商品/供应链影响] → [行业影响] → [持仓标的影响]
- **盲点提示**：[容易被忽视的风险因素]
```

### D4: 文件位置

**选择**：模板放在 `skills/ymos-core/templates/geopolitical-analysis.md`（共享模板），由 P13 prompt 引用

**理由**：地缘政治分析可能被其他 skill 引用（如 P3 Event Impact），放在 core 作为共享资源。

## Risks / Trade-offs

- **[LLM 地缘政治知识可能过时]** → 模板要求"历史模式"而非"最新情报"，降低对实时数据的依赖
- **[传导路径分析可能过于简化]** → 通过要求"至少列出 2 级传导"来保证深度
- **[模板激活噪音]** → 非地缘政治事件被误触发 → 关键词域粗筛 + LLM 细判断双层过滤
