## Context

YMOS 的财报分析能力分散：
- ymos-radar 可以检测财报发布事件
- ymos-research 的 P4 可以分析财报影响
- 但缺乏标准化的财报更新报告

投资者需要：
1. 快速了解财报表现（Beat/Miss）
2. 详细的财务数据对比
3. 可视化图表
4. 完整的数据来源引用

Financial Services 的 Earnings Analysis 提供了：
- 8-12 页机构级报告
- 3000-5000 字详细分析
- 8-12 张图表
- 完整的数据来源引用

YMOS 需要建立类似但更适合个人投资者的财报报告能力。

## Goals / Non-Goals

**Goals:**
- 建立财报更新报告生成能力
- 提供 Beat/Miss 标准化分析
- 提供完整的数据来源引用（带超链接）
- 支持图表生成
- 提供专业级报告模板

**Non-Goals:**
- 不提供实时财报推送（手动触发）
- 不实现复杂的财务建模（聚焦财报分析）
- 不替代专业分析师的深度研究

## Decisions

### 1. 报告结构

**决策：** 采用简化版机构报告结构

**结构：**
1. **执行摘要** — 1 段话总结财报表现
2. **关键数据** — 收入、利润、EPS 等核心指标
3. **Beat/Miss 分析** — 与预期对比
4. **分部/业务线分析** — 如果适用
5. **前瞻指引** — 管理层指引变化
6. **估值影响** — 对估值的影响
7. **数据来源** — 完整引用列表

**理由：**
- 保留核心内容，去除冗余
- 适应个人投资者的阅读习惯
- 便于快速获取关键信息

### 2. Beat/Miss 分析

**决策：** 标准化的 Beat/Miss 分析格式

**格式：**
```markdown
| 指标 | 实际值 | 预期值 | 差异 | 差异% |
|------|--------|--------|------|-------|
| 收入 | 120M | 115M | +5M | +4.3% |
| EPS | 1.2 | 1.15 | +0.05 | +4.3% |
```

**理由：**
- 表格格式直观易读
- 百分比差异便于比较
- 与 Financial Services 一致

### 3. 数据源引用

**决策：** 强制性数据源引用，带超链接

**引用格式：**
```markdown
**数据来源：**
- 财报发布：[公司名称 QX 202X 财报](URL)
- 10-Q/10-K：[SEC EDGAR](URL)
- 财报电话会议：[Seeking Alpha/公司官网](URL)
- 投资者演示：[公司官网](URL)
- 预期数据：[Bloomberg/FactSet](URL)
```

**理由：**
- 提供数据可追溯性
- 便于用户验证
- 符合机构标准

### 4. 图表生成

**决策：** V1 不支持图表，V2 使用 Python 生成

**V1（当前）：**
- 使用 Markdown 表格展示趋势数据
- 用户可自行将数据导入 Excel 生成图表

**V2（未来）：**
- 使用 matplotlib/plotly 生成图表
- 图表类型：收入趋势、利润率趋势、EPS 趋势
- 图表嵌入报告

**理由：**
- 分阶段实现，降低初始复杂度
- V1 提供数据，V2 提供可视化
- 与 excel-output skill 集成

### 5. 前瞻指引分析

**决策：** 对比当前指引与之前指引

**格式：**
```markdown
**管理层指引：**
- 收入：[范围] vs. 上期 [范围]
- 利润率：[范围] vs. 上期 [范围]
- 其他关键指标：[变化]
```

**理由：**
- 指引变化是股价重要驱动因素
- 标准化格式便于对比

### 6. 估值影响分析

**决策：** 简化版估值影响分析

**内容：**
- 基于 DCF 或可比公司的估值影响
- 与当前市价对比
- 简要的买入/持有/卖出建议（不作为投资建议）

**理由：**
- 提供估值参考
- 保持简洁，不做详细建模
- 明确声明不是投资建议

### 7. 报告格式

**决策：** Markdown 格式为主，未来支持 Word

**理由：**
- Markdown 易于版本控制和协作
- 与 YMOS 现有格式一致
- 可转换为其他格式（如 PDF）

### 8. 与现有 Skill 集成

**决策：** 创建独立的 `ymos-earnings-update` skill

**集成点：**
- `ymos-radar`: 检测到财报事件时，提示生成财报报告
- `ymos-strategy`: 分析买入/卖出时引用财报分析结果

**理由：**
- 独立 skill 保持可测试性
- 可选调用，不影响现有流程

## Risks / Trade-offs

### Risk 1: 数据准确性

**风险：** 财报数据可能存在错误或过时

**缓解：**
- 强制引用官方数据源
- 提示用户验证关键数据
- 记录数据获取日期

### Risk 2: 预期数据来源不一致

**风险：** 不同数据源（Bloomberg/FactSet）的预期值可能不同

**缓解：**
- 明确标注预期数据来源
- 提供多个数据源的对比（如可用）
- 允许用户手动输入预期值

### Risk 3: 报告复杂度

**风险：** 机构级报告可能过于复杂，普通投资者难以消化

**缓解：**
- 提供执行摘要快速了解核心信息
- 使用清晰的章节结构
- 提供关键数据高亮

### Risk 4: 实时性问题

**风险：** 财报报告生成可能滞后于市场反应

**缓解：**
- 明确标注报告生成时间
- 强调报告是分析工具，非交易信号
- 建议用户结合最新价格

## Open Questions

1. **图表支持：** V1 是否完全不提供图表？
   - 建议：V1 提供数据表格，V2 实现图表生成

2. **历史对比：** 是否需要与历史财报对比？
   - 建议：提供近 4 个季度的趋势数据

3. **同行对比：** 是否需要与同行财报对比？
   - 建议：V2 功能

4. **报告长度：** 多长的报告合适？
   - 建议：3-5 页（简化版），8-12 页（完整版）

5. **多语言：** 是否需要支持英文报告？
   - 建议：当前只支持中文，未来扩展

6. **自动化：** 是否需要自动生成所有持仓的财报报告？
   - 建议：手动触发，避免信息过载

7. **归档：** 历史报告如何管理？
   - 建议：按年份/季度归档，保留最近 2 年
