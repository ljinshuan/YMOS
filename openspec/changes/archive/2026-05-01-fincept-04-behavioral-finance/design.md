## Context

YMOS 的 P12 Referee 是所有策略决策的最终纪律关卡，当前基于"死亡边界"规则清单做零容忍扫描，覆盖杠杆、情绪交易、风格漂移等红线。这是规则层面的纪律执行，但缺少认知层面的偏误检测——用户可能没有违反明确规则，但正在被系统性认知偏误驱动。

FinceptTerminal 的 behavioral_finance_analyst 是独立 agent。对于 YMOS，更好的方式是将行为金融学框架融入现有 P12，而非新建独立 prompt，因为 P12 天然是偏误检测的最佳位置。

## Goals / Non-Goals

**Goals:**
- 在 P12 中增加结构化的行为偏误扫描模块
- 覆盖投资决策中最常见的 8 大认知偏误
- 每个偏误配备可操作的自检问题（而非抽象定义）
- 输出"行为偏误风险评级"，与现有 P12 verdict 共存

**Non-Goals:**
- 不做独立的行为金融 prompt 或 skill
- 不对用户做心理画像
- 不改变 P12 的"零容忍"哲学

## Decisions

### D1: 8 大核心偏误选择

**选择**：

| 偏误 | 投资中的表现 | 自检问题示例 |
|---|---|---|
| 处置效应 | 过早止盈、死扛亏损 | "如果现在空仓，你会在这个价格买入吗？" |
| 锚定效应 | 以买入价/历史高点为参考做决策 | "你的目标价基于什么？是基本面还是历史价格？" |
| 损失厌恶 | 对亏损的恐惧大于对盈利的渴望 | "你是在避免亏损还是在追求收益？" |
| 过度自信 | 高估自己的信息优势或判断能力 | "你的分析中有多少是事实，多少是推测？" |
| 确认偏误 | 只寻找支持自己观点的信息 | "你最近主动找过反对观点吗？" |
| 近因偏差 | 过度重视最近发生的事件 | "如果这个新闻是三个月前的，你还会有同样的反应吗？" |
| 羊群效应 | 跟随市场情绪或他人操作 | "你的逻辑独立于市场共识吗？" |
| 沉没成本 | 因为已经投入而不愿止损 | "如果今天第一次看到这只股票，你会买吗？" |

**理由**：这 8 个覆盖了投资决策链中最常出现的认知陷阱，与 YMOS 已有的 P5（FOMO）和 P12（规则）形成互补。

### D2: 集成方式

**选择**：在 P12 prompt 的现有结构中，在"死亡边界扫描"之后、verdict 之前，插入"行为偏误扫描"section

**理由**：偏误检测是规则检查的自然延伸。先查规则红线，再查认知偏误，最后给综合 verdict。

### D3: 输出格式

**选择**：P12 输出增加以下字段：
- `行为偏误风险`：高/中/低/无
- `触发偏误`：列出命中的偏误名称（0-N 个）
- `自检问答`：对命中的偏误，输出对应的自检问题和用户的决策逻辑

**理由**：与 P12 现有的 verdict（pass/reject）格式保持一致风格。

### D4: 案例库补充

**选择**：在 `diagnosis_case_library.md` 中新增"行为偏误"案例类别，补充 3-5 个真实案例

**理由**：案例库已有 5 个类别，新增"行为偏误"使覆盖更完整，也为 ymos-diagnosis 的问诊模式提供参考素材。

## Risks / Trade-offs

- **[P12 输出变长]** → 偏误扫描增加 ~500-800 Token → 可接受，P12 是关键节点，多一点分析值得
- **[误报风险]** → LLM 可能过度标记偏误 → 通过要求"仅在有明确证据时标记"来控制
- **[与 P5 重叠]** → P5 FOMO Killer 和行为偏误有交叉（如 FOMO ≈ 损失厌恶+羊群效应）→ P5 聚焦"该不该买"，P12 聚焦"你的决策过程是否健康"，维度不同
