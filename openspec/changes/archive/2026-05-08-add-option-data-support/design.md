## Context

目前 YMOS 通过富途 OpenD 的 `get_derivative_unusual` API 获取衍生品异常数据，该 API 返回的是"已发生"的异常信号（如大额期权交易、波动率突升等）。但对于主动交易决策，还需要"当前状态"数据：完整的期权链分布、隐含波动率（IV）曲面、Put/Call Ratio（PCR）、希腊值分布等。

富途 OpenD 提供了以下期权相关 API：
- `get_option_expiration_date(code)`: 获取所有到期日
- `get_option_chain(code, ...)`: 获取期权合约列表（支持按日期/类型/价内外筛选）
- `get_market_snapshot([codes])`: 获取期权实时数据（价格、IV、希腊值、未平仓）

这些 API 可以构建完整的期权数据视图，而不仅仅是异常信号。

## Goals / Non-Goals

**Goals:**
1. 新增 `ymos fetch-option-chain` CLI 命令，获取完整期权链数据
2. 期权数据作为**可选**分析依据，集成到投资雷达和策略分析
3. 支持按到期日、期权类型、行权价区间筛选
4. 输出标准化的 JSON 格式，便于后续分析和 prompt 处理

**Non-Goals:**
1. 不提供期权交易建议（仅提供数据和分析）
2. 不实现期权定价模型（使用富途提供的 IV 和希腊值）
3. 不覆盖所有期权市场（仅美股通过富途 OpenD）

## Decisions

### 1. 新增独立 data source 模块

**决策**：创建 `cli/core/sources/option_chain.py` 作为期权数据源，复用现有的 `futu_utils` 连接管理。

**理由**：
- 与现有 `derivatives_anomaly.py`、`futu_quote.py` 等数据源模块保持一致
- 便于后续复用（如投资雷达、策略分析）

**替代方案**：
- 直接在 CLI 命令中实现：会导致代码重复，不利于复用

### 2. 数据分两次获取

**决策**：先获取期权链静态数据（`get_option_chain`），再批量获取实时数据（`get_market_snapshot`）。

**理由**：
- `get_option_chain` 返回完整的合约列表（365+ 个合约），但缺少实时价格和希腊值
- `get_market_snapshot` 可以同时获取多个期权的实时数据（需要先订阅 QUOTE）
- 分离静态和动态数据，便于缓存和增量更新

**替代方案**：
- 单次 API 获取：富途 API 不支持，必须两次调用

### 3. 输出 JSON 包含原始数据和派生指标

**决策**：输出 JSON 包含：
- 原始数据：富途 API 返回的所有字段（便于调试和后续扩展）
- 派生指标：IV 分位数、PCR（Put/Call Ratio）、OI 变化率（便于分析）

**理由**：
- 原始数据保证完整性，派生指标直接可用

### 4. 可选集成策略

**决策**：期权数据作为**可选**分析依据，OpenD 不可用时优雅降级。

**理由**：
- 期权数据不是所有用户都需要（如纯价值投资者）
- OpenD 连接可能不稳定，不应阻塞主流程

### 5. 投资雷达新增独立 section

**决策**：在桥接报告中新增「期权市场情绪」独立 section，不与现有技术分析/资金流 section 混合。

**理由**：
- 期权分析是独立维度，保持清晰分离
- 便于用户选择性关注

## Risks / Trade-offs

### [Risk] 期权合约数量大，API 调用可能超时

**缓解措施**：
- 限制单次获取的到期日范围（如最近 1-2 个月）
- 支持按行权价区间筛选（如 ATM ±20%）
- 增加超时重试机制

### [Risk] OpenD 权限问题

**缓解措施**：
- API 调用失败时返回明确的错误信息（权限不足/未开通）
- 在 CLI 文档中说明期权数据需要开通相应权限

### [Risk] IV 和希腊值数据缺失（流动性差期权）

**缓解措施**：
- 输出中标记数据缺失的期权
- 分析时仅使用有效数据（如 IV > 0 的期权）

### [Trade-off] 数据新鲜度 vs API 调用频率

- 期权数据变化较快，但高频调用可能被限流
- **方案**：雷达运行时每次获取最新数据，不缓存
