# 进阶指南

> YMOS 帮你从投研的重复劳动中解放出来——数据收集、信号追踪、报告生成交给 AI，
> 你专注于三件最重要的事：**打磨策略**、**做关键判断**、**深度调研**（自己读研报、挖行业，把材料拖入个股文件夹供 AI 后续分析）。

> 开箱即用只需 RSS（零 API Key）。以下是让系统能力持续进化的路径。

## Level 1：Finnhub Free API Key（推荐）

### 它能带来什么

| 能力 | 无 Finnhub Key | 有 Finnhub Key |
|:---|:---|:---|
| 美股/Crypto 价格 | Yahoo Finance 回退 | Finnhub 实时报价 + Yahoo 互备 |
| 英文新闻 | 仅 RSS 源 | + Finnhub 持仓匹配新闻 |
| P15 触发检测 | 无 | 自动标记财报/并购等关键事件 |
| 个股新闻 | 无 | `company-news` 按 Ticker 精准拉取 |

### 如何获取

1. 访问 [finnhub.io](https://finnhub.io/) 注册免费账号
2. 在 Dashboard 中复制你的 API Key
3. 编辑 `.env` 文件：
   ```
   FINNHUB_API_KEY=你的key
   ```

### 验证

```bash
# 测试价格查询
python3 Eyes/scripts/fetch_price_api.py --symbols "AAPL,NVDA" --output /tmp/test_price.json

# 测试新闻拉取
python3 Eyes/scripts/fetch_finnhub_news.py --hours 24 --output /tmp/test_news.json
```

### 免费 Key 的限制

- 每分钟 60 次 API 调用
- 仅覆盖美股和 Crypto（A股/港股不支持）
- 行情数据有 15 分钟延迟
- 个股新闻窗口最长 1 年

> 对于 YMOS 的使用场景（日频扫描 + 事件触发），免费 Key 完全够用。

## Level 1+：Tushare Token（A 股用户推荐）

### 它能带来什么

| 能力 | 无 Tushare | 有 Tushare |
|:---|:---|:---|
| A 股价格（.SS/.SZ） | Yahoo Finance 回退（延迟高、偶尔不稳定） | Tushare 实时报价 + Yahoo 互备 |
| A 股日线数据 | 无 | 支持历史日线查询 |

### 如何获取

1. 访问 [tushare.pro](https://tushare.pro/) 注册账号
2. 在个人中心获取 Token
3. 编辑 `.env` 文件：
   ```
   TUSHARE_TOKEN=你的token
   ```

### 免费 Token 的限制

- 注册并完善个人信息后获得 120 积分（不消耗），达到日常接口调用的使用门槛
- 部分高级接口需要更高积分等级
- 仅覆盖 A 股（.SS/.SZ），美股/港股/Crypto 走 Finnhub 或 Yahoo

### 三源价格路由机制

YMOS 的价格扫描自动分流（`Eyes/scripts/fetch_price_router.py`）：

```
Ticker 后缀判断：
  .SS / .SZ → Tushare（无 Token 时回退 Yahoo）
  .HK       → Yahoo Finance
  其他       → Finnhub（无 Key 时回退 Yahoo）
```

> Finnhub 和 Tushare 可独立配置，互不影响。建议：美股用户配 Finnhub，A 股用户配 Tushare，两市都看就都配。

## Level 1.5：Futu OpenD（富途证券用户推荐）

### 它能带来什么

YMOS 通过 Futu OpenD（富途证券本地网关）获取 Broker 级数据，覆盖港股/A股/美股：

| 能力 | 命令 | 说明 |
|:---|:---|:---|
| 真实持仓 | `ymos position fetch` | 自动导入持仓到状态机，入职时省去手动录入 |
| 交易记录 | `ymos trade-history fetch --days 30` | P12 纪律审查引用实际交易行为，检测追涨杀跌等偏误 |
| 个股情绪 | `ymos fetch-sentiment --ticker AAPL` | 评论情绪分析，辅助策略判断 |
| 资金异动 | `ymos fetch-capital-flow fetch` | 投资雷达中的资金流异常信号 |
| 选股筛选 | `ymos screen --market HK` | 按市场/条件筛选标的 |
| 技术异动 | `ymos fetch-technical-anomaly` | 技术面异常信号检测 |
| 衍生品异动 | `ymos fetch-derivatives-anomaly` | 期权/期货异动检测 |

### 与其他模块的关系

| 模块 | 如何使用 Futu 数据 |
|:---|:---|
| ymos-onboarding | 自动导入持仓，省去手动录入 |
| ymos-strategy | P12 纪律审查引用交易记录作为行为偏误实证 |
| ymos-radar | 资金异动信号纳入 Tier 1/Tier 2 事件评级 |
| ymos-reconcile | 真实持仓 vs 状态机一致性校验 |
| ymos-diagnosis | Phase 2B 情绪审计引用交易记录分析行为模式 |

### 安装步骤

1. **开立富途证券账户**（已有则跳过）
   - 访问 [futunn.com](https://www.futunn.com/) 开户
   - 支持港股、A股（沪港通/深港通）、美股

2. **下载 Futu OpenD**
   - GitHub: [FutunnOpen/py-futu-api/releases](https://github.com/FutunnOpen/py-futu-api/releases)
   - macOS / Windows / Linux 均支持

3. **启动并登录**
   ```bash
   # 解压后运行
   ./FutuOpenD
   # 浏览器自动打开，扫码登录富途账户
   # 默认监听 127.0.0.1:11111
   ```

4. **验证**
   ```bash
   uv run ymos position fetch
   # 成功返回持仓数据 → 配置完成
   ```

### 注意事项
- Futu OpenD 需保持运行状态（后台进程即可）
- **所有 Futu 相关命令在 OpenD 离线时自动跳过，不阻塞 YMOS 核心功能**
- 不使用富途证券的用户可跳过，YMOS 仍可完整运行
- 无需 API Key，无需 .env 配置，只需本地运行 OpenD 即可

## RSS 信息源自定义

### 当前默认源

YMOS 内置 15 个免费 RSS 源，分为 5 个分类：

| 分类 | 数量 | 主要源 |
|:---|:---|:---|
| 美股 | 3 | Bloomberg Markets, CNBC Markets, FT Markets |
| 宏观 | 2 | CNBC Finance, The Economist |
| 科技 | 5 | Bloomberg Tech, FT Tech, TechCrunch, Nvidia Blog, The Verge |
| Crypto | 1 | Bloomberg Crypto |
| 深度洞察 | 4 | Seeking Alpha, Stratechery, Bloomberg Opinion, TechInAsia |

### 如何自定义

编辑 `Eyes/scripts/rss_sources.json`，按以下格式增减源：

```json
{
  "name": "你的源名称",
  "url": "https://example.com/feed/",
  "category": "美股",
  "priority": "medium",
  "note": "简要说明"
}
```

### 补充 RSS：与 Market Data API 并行使用

如果你已经配置了 Market Data API（Level 2），但还想额外订阅一些 API 覆盖不到的 RSS 源（如特定行业博客、独立分析师），可以创建补充 RSS 配置：

1. 复制 `Eyes/scripts/rss_sources_custom.json.example` 为 `rss_sources_custom.json`
2. 填入你的自定义 RSS 源（格式与 `rss_sources.json` 一致）
3. 市场洞察 SOP 会自动检测此文件，存在即拉取，不存在即跳过

**补充 RSS 与主数据源独立运行**——无论你用 API 还是默认 RSS 作为主数据源，补充 RSS 都会额外拉取并作为 P13 分析的补充输入。

```bash
# 补充 RSS 的调用方式（SOP 自动处理，通常不需要手动执行）
python3 Eyes/scripts/fetch_rss.py 1 \
  --config Eyes/scripts/rss_sources_custom.json \
  --output "Eyes/市场洞察/Raw_Data/YYYY-MM/supplementary_rss_YYYYMMDD.json"
```

## Level 3：更多金融数据 API（自定义集成）

YMOS 的脚本层是模块化的——每个数据源一个独立脚本，统一输出 JSON。你可以参考现有的 `Eyes/scripts/fetch_finnhub_news.py` 或 `Eyes/scripts/fetch_price_api.py` 的模式，接入任何金融数据 API。

核心模式很简单：**一个端点 + 一个 API Key → 输出 JSON → 喂给 SOP 处理链**。

### 推荐的金融数据服务商

| 服务商 | 免费层 | 主要能力 | 适合场景 |
|:---|:---|:---|:---|

| **Polygon.io** | 有限 | 美股实时/历史行情、期权链 | 精准行情、期权策略（P10） |
| **Benzinga Pro API** | 付费 | 实时新闻、评级变动、内部交易 | 高频事件驱动 |

### 如何接入新数据源

1. **创建脚本**：`Eyes/scripts/fetch_[source_name].py`
   - 参考 `Eyes/scripts/fetch_finnhub_news.py` 的结构
   - 从 `.env` 读取 API Key
   - 输出标准 JSON 到 `Eyes/` 对应目录

2. **更新 `.env.example`**：
   ```
   TIINGO_API_KEY=
   ALPHA_VANTAGE_KEY=
   ```

3. **更新 SOP**：在对应 SOP 中添加可选的数据拉取步骤（参考 `Eyes/SOP_市场洞察.md` 的 Step 2 模式）

4. **Agent 自动识别**：只要脚本存在且 API Key 配置了，SOP 步骤会自动执行

## 投资策略诊断：审视你的投资逻辑

> 在优化策略之前，先确认你的投资逻辑本身没有问题。

YMOS 内置了一个独立的投资策略诊断模块（`Brain/ymos-diagnosis/`），它不做策略制定、不做交易执行，只做一件事：**帮你审视整体投资逻辑和系统健康度**。

### 什么时候用

| 场景 | 推荐模式 |
|:---|:---|
| 刚接触 YMOS，不确定自己的投资策略是否合理 | 体检模式 → 梳理清楚再填写投资偏好 |
| 连续亏损或大幅回撤后 | 体检模式 → 检查是策略问题还是执行问题 |
| 对某个具体投资决策拿不准 | 问诊模式 → 先判断问题本身成不成立 |
| 每季度定期审视 | 体检模式 → 7 项检验，看看有没有新的漏洞 |
| 策略调整前 | 体检模式 → 确认调整方向是否合理 |

### 怎么用

让 AI 读取诊断入口文件：

```
请读取 [你的YMOS路径]/Brain/ymos-diagnosis/SKILL.md

诊断一下我的策略
```

AI 会引导你选择问诊或体检，然后一步步带你走完。

### 和 P 系列提示词的区别

| | 诊断模块 | P 系列提示词 |
|:---|:---|:---|
| **做什么** | 审视整体投资逻辑 | 执行具体投资分析 |
| **关注点** | 你的系统有没有漏洞 | 这只股票该不该买/卖 |
| **产出** | 诊断报告 / 问题消解 | 策略报告 / 买卖建议 |
| **频率** | 按需 / 定期 | 每日 / 事件驱动 |

两者互补：诊断模块帮你确认**方向对不对**，P 系列帮你**走对的路**。

> 诊断模块也作为独立开源项目发布：[github.com/Evan-XYZ/ymos-diagnosis](https://github.com/Evan-XYZ/ymos-diagnosis)，不需要安装 YMOS 也能单独使用。

## 策略进化：P 系列提示词定制

> 这是最核心的进阶——不是接更多数据源，而是**让大脑变聪明**。

### P 系列提示词是什么

`.claude/skills/<skill>/prompts/ 或 .claude/skills/ymos-core/prompts/` 下的 P1-P16 提示词是 YMOS 的投资决策框架。它们代表一种经过验证的投资思路和决策路由：

| 阶段 | 提示词 | 职责 |
|:---|:---|:---|
| **建档** | P1 Genesis | 基石档案（治理/壁垒/财务/周期） |
| **追踪** | P4 Radar | 五维度关注点雷达 |
| **定性** | P2 Phase Check | PVE/PVP 阶段判断 |
| **事件** | P3 Event Impact | 事件冲击分析 |
| **买入** | P5 FOMO Killer | 买入前审计 |
| **持有** | P6 Profit Keeper | 持仓评估 |
| **组合** | P7 Portfolio Check | 仓位再平衡 |
| **宏观** | P8 Macro Filter | 宏观压力测试 |
| **估值** | P9 Valuation | 反向 DCF 估值 |
| **期权** | P10 Options | 期权策略 |
| **复盘** | P11 Autopsy | 交易复盘 |
| **纪律** | P12 Referee | 纪律审查（最终关卡） |
| **市场** | P13 Market Scan | 市场全景扫描 |
| **板块** | P14 Sector | 热门板块挖掘 |
| **深度** | P15 Deep Insight | 事件深度洞察 |
| **财报** | P16 Earnings | 财报深度解读 |

### 如何定制

P 系列提示词是**框架样板**。真正的进阶是结合你自己的投资策略来改造它们：

1. **修改权重和判断标准**：比如 P5 FOMO Killer 中的评分阈值
2. **增加行业特定逻辑**：比如 P1 中加入你熟悉行业的特殊指标
3. **调整风险偏好**：P12 纪律审查中的红线规则
4. **增加新的 P 模块**：遵循相同命名规范，SOP 中增加路由即可

> ⚠️ 修改提示词前，建议先用原版跑通几轮完整流程，理解每个 P 模块在决策链中的角色。

### SOP 路由定制

五大策略路由（买入/加仓/持有/卖出/再平衡）定义在 `Brain/SOP_策略分析.md` 中。你可以：
- 调整每条路由的 P 链顺序
- 增加条件分支（如特定市值区间走不同路由）
- 增加新的路由类型

## Agent Skills 与跨模块联动

### Agent 能力自检

YMOS 设计为可被不同 Agent 编排产品驱动（Claude Code、ChatGPT Codex、Google Antigravity、OpenClaw 等）。Agent 进入 YMOS 时，会自动检查四类能力：

| 能力类别 | 检查内容 | 影响的环节 |
|:---|:---|:---|
| **Web 搜索** | 能否实时搜索互联网？ | P1 建档、P2 阶段判断、P3 事件分析 |
| **工具/Skills** | 是否挂载了 MX_Skills 或其他数据工具？ | 选股筛选、财务数据、宏观数据 |
| **文件系统范围** | 能否读取 YMOS 同级目录（如 BrainStorm 等）？ | 上下文补充、跨模块联动 |
| **代码执行** | 能否运行 Python 脚本？ | 数据拉取（Eyes/scripts/*.py） |

有什么能力就用什么能力——缺少某项能力不会阻塞主流程，只是分析质量会有差异。

### Agent 可增强的环节

以下是 Agent 的额外能力可以提升质量的具体环节：

| SOP 环节 | 增强方式 | 约束 |
|:---|:---|:---|
| **P1 Genesis（建档）** | 用搜索补充公司财务、竞争格局、行业数据 | 不改变 P1 模板结构 |
| **P2 Phase Check** | 搜索最新价格/资金流数据辅助判断 | 不改变 PVE/PVP 判断标准 |
| **P3/P15 事件分析** | 搜索最新新闻、公告替代手动输入 | 不改变分析框架 |
| **P13 市场扫描** | 若有财经搜索工具 → 补充中文财经资讯 | 不替代 RSS 主链，作为补充 |
| **选股筛选** | 若有选股工具 → P13/P14 发现主题后自动筛选候选池 | 输出须转入 P1 建档流程 |
| **财务快照** | 若有财务数据工具 → P1 建档时批量拉取指标 | 写入个股基础知识库 |
| **宏观数据** | 若有宏观工具 → P8 压力测试前获取最新数据 | 使用标准化格式 |

### 系统架构比喻

- **大脑**：Brain/（P 系列提示词 + 策略路由）— 做决策、出结论
- **眼睛**：Eyes/（市场洞察 + 投资雷达 + 价格扫描）— 看市场、报信号
- **记忆**：持仓与关注/（状态机 + 个股文件夹）— 存状态、积累上下文
- **手脚**：Skills 工具（财经搜索、Web 搜索等）— 获取数据、执行查询
- **感官增强**：同级知识模块（BrainStorm 等）— 打破数据墙，丰富上下文

### 跨模块联动

如果你在同一 Vault 中维护了其他知识模块（如头脑风暴/BrainStorm、研究笔记等），Agent 可以读取这些模块作为分析背景——例如用 BrainStorm 中的投资洞察丰富 P13 市场扫描的上下文。

| 模块类型 | 联动方式 | 典型场景 | 限制 |
|:---|:---|:---|:---|
| **Skills 工具（手脚）** | SOP 步骤按需调用 | 财经搜索/财务数据/选股/宏观数据补充 | 调用失败不阻断主流程 |
| **知识模块（感官增强）** | Agent 读取作为上下文 | BrainStorm 投资洞察 → P13 背景 | 只读引用，不修改源模块 |

### 进化边界

| 项目 | 规则 |
|:---|:---|
| P 系列提示词 | **可进化**，但改动需人工审阅后生效 |
| Human in the Loop（P5/P6/P12） | **不可绕过** — 买卖决策必须人工确认 |
| 状态机写入 | **必须通过 SOP 路径** — 不可直接编辑 |
| 报告命名规则（同日覆盖） | **固定** — 系统一致性依赖 |

### 扩展新模块（标准流程）

想要接入新的数据源或分析能力？标准 5 步：

1. **数据脚本** → `Eyes/scripts/fetch_xxx.py`（输出 JSON）
2. **处理提示词** → `.claude/skills/<skill>/prompts/ 或 .claude/skills/ymos-core/prompts/xxx.md`（如需要）
3. **SOP 步骤** → 在对应 SOP 中添加可选步骤
4. **环境变量** → 更新 `.env.example`
5. **文档** → 更新 `进阶指南.md` + `README.md`

> Agent 的完整权限表、文件读写规则和模块依赖图详见 `AGENT_GUIDE.md`。

## 定时任务设置

### 日常四步闭环

YMOS 的核心日常工作流是四步依次执行：

```
1. 跑一下市场洞察     → 拉取数据 + P13 分析         ~3-5 分钟
2. 跑一下投资雷达     → 趋势回顾 + 价格扫描 + 建议   ~5-8 分钟
3. 跑一下策略分析     → 消费雷达建议 + 策略路由       ~10-15 分钟/标的
4. 收口一下           → 一致性校验 + 持仓备忘录视图   ~2-3 分钟
```

> ⚠️ **依赖关系**：2 依赖 1 的市场洞察输出，3 依赖 2 的投资雷达建议，4 校验 3 的写回并生成人读层执行看板。必须按顺序执行，每步间隔 5-15 分钟确保前一步完成。

### 推荐时间窗口

| 场景 | 市场洞察 | 投资雷达 | 策略分析 | 持仓收口 | 适用说明 |
|:---|:---|:---|:---|:---|:---|
| **A股盘中** | 10:30 | 10:45 | 11:00 | 11:15 | 盘中捕捉异动，及时响应 |
| **美股盘前** | 20:30 | 20:45 | 21:00 | 21:15 | 美股开盘前做好功课 |
| **收盘后复盘** | 收盘后 30min | +15min | +15min | +15min | 复盘当日行情，规划明日 |

你可以根据自己的作息和投资习惯调整时间。核心原则：**在你有时间审阅报告的时段之前跑完**。

### 怎么设置

把 SOP 路径和指令告诉 AI，让它帮你配置。以 Claude Code 为例：

```
请读取 /你的路径/YMOS/总入口暗号.md

帮我设置四个定时任务：
1. 每天 10:30 执行「跑一下市场洞察」
2. 每天 10:45 执行「跑一下投资雷达」
3. 每天 11:00 执行「跑一下策略分析」
4. 每天 11:15 执行「收口一下」
```

> 💡 **务必带上 `总入口暗号.md` 的完整路径**。Agent 需要通过这个文件定位你的 YMOS 目录和所有 SOP。没有记忆功能的 Agent（如无持久化 memory 的对话窗口）每次执行都需要这个路径作为入口。

### 策略分析的自动调研

定时触发 `跑一下策略分析` 时，除了消费"策略分析建议"，还会自动消费投资雷达的"调研建议"：
- 如果某标的缺少 P1/P4、信息过期（>30天）、或新关注未建档，策略分析会先调用初始调研 SOP 补全
- 补全后再进入策略路由，确保分析有完整的个股上下文
- 用户不需要手动 `调研一下 TICKER`——定时任务链会自动处理

### 进阶：多角色 Agent 编排

如果你使用支持 subagent / 多角色编排的 Agent 平台（如 OpenClaw），可以更进一步：把定时任务链中的不同 SOP 分配给不同的 Agent 角色，各角色有独立的读写权限和职责边界：

- **Market Insight Agent**：只负责看市场，产出市场洞察
- **Investment Radar Agent**：把市场信号翻译成"跟我有关的"桥接报告
- **Strategy Agent**：消费雷达建议，执行策略路由
- **Portfolio State Agent**：统一状态写回，维护持仓/Watchlist 真相源

这种编排让每个角色的职责更清晰、权限更收敛，也更容易调试和迭代。这属于"养系统"的范畴——跑通基础闭环之后，根据你的使用习惯逐步优化编排方式。

> 多角色编排不是必须的。单 Agent + 定时任务链已经能完整运行 YMOS 的所有功能。编排是锦上添花，适合有深度使用需求的用户自行探索。

> 对 OpenClaw 不熟悉的用户，通过 Claude 或 ChatGPT 的订阅会员产品也可以完整使用 YMOS。

### 模型选择：能用最好的就用最好的

YMOS 的核心价值不在脚本，而在 **SOP + P 系列提示词的分析质量**——这完全取决于底层模型的能力。模型强 1%，投资判断就可能好 1%，而在投资里，1% 的差距可以是决定性的。

**作者建议（2026 年 3 月）**：当前最强的通用模型是 **Claude Opus 4.6** 和 **ChatGPT-5.4**。如果你的 Agent 工具支持切换模型，永远选当下最强的那个。

不要因为省 token 而降级模型。对投资来说，模型表现好 1%，对胜率的提高都不一样——视角的补充更全面，策略的制定更严谨，风险的识别更敏锐。省下来的 token 成本，远不如一次误判的代价。

如果资源有限，**优先把最好的模型用在策略分析环节**（Brain 模块的 P5/P6/P8/P9/P12 等），这里的判断质量对最终投资决策影响最大。市场洞察和投资雷达可以用稍弱的模型，但策略分析和纪律审查一定要用最强的。

## 系统扩展性设计理念

YMOS 提供的是：

1. **开箱即用的最小闭环**：纯 RSS + Yahoo Finance，零配置
2. **完整的投资思路框架**：P1-P16 覆盖从建档到复盘的全流程
3. **持仓/Watchlist 管理框架**：状态机 + 个股文件夹 + 自动化 P 链
4. **可魔改的扩展接口**：
   - 数据层（Eyes/scripts/）：增加任何 API 数据源（价格/新闻/基本面）
   - 处理层（.claude/skills/<skill>/prompts/ 或 .claude/skills/ymos-core/prompts/）：自定义 CIO 提示词、修改 P 系列提示词
   - 路由层（各 SOP）：增加新的暗号和工作流
   - 展示层：所有产出物是 Markdown，可接入任何展示工具

### 可视化看板（Dashboard）

持仓收口 SOP 的最后一步会自动生成一个可视化 Dashboard（HTML 文件），综合展示当日的市场洞察、投资雷达、持仓状态和待办事项。浏览器打开即可查看，无需额外工具。

**输出路径**：`持仓与关注/dashboard/YYYY-MM/dashboard_YYYY-MM-DD.html`（按月归档，每日一个文件）

Dashboard 由 Agent 根据 SOP 中的布局要求直接生成（软路由），不依赖固定的脚本解析——即使你修改了报告格式，Agent 也能正确理解内容并渲染看板。

如果你希望进一步定制看板样式或集成到自己的前端系统，可以基于生成的 HTML 做二次开发。所有数据来源都是本地 Markdown 文件，没有外部依赖

5. **策略进化的核心载体**：P1-P16 提示词是框架样板——它代表一种经过验证的投资思路和决策路由。
   真正的进阶不只是接更多数据源，而是**结合你自己的投资策略，改造 P 系列提示词和 SOP 路由**。
   这是系统大脑进化的核心。

**你的系统，你做主。** 框架搭好了，能力升级是渐进的——技术配置让手脚更灵活，策略进化让大脑更聪明。
