# Dashboard 布局和样式要求

### 数据来源

你在 Step 1-3 已经读取了所有需要的文件。无需重新读取。

### 输出

生成一个**自包含 HTML 文件**（CSS 内嵌，无外部依赖），写入：

```
data/dashboard/YYYY-MM/dashboard_YYYY-MM-DD.html
```

### 布局要求

生成的 HTML 须包含以下区块，从上到下排列：

**1. 顶栏**
- 左侧：`YMOS │ 投资操作系统`
- 右侧：当日日期

**2. 状态概览（卡片）**
- 左侧：状态灯（🟢 无待办 / 🟡 有待确认 / 🔴 有紧急动作）+ 状态文字
- 右侧：持仓数量

**3. 每日工作流（卡片）**
- 4 个步骤横向排列：市场洞察 → 投资雷达 → 策略分析 → 持仓收口
- 已完成的步骤：绿色背景 + ✅ + "今天已完成"
- 未完成的步骤：灰色背景 + 序号 + "点击复制指令"
- 底部：`已完成 N/4`

**4. 战略简报（卡片，来自市场洞察）**
- 市场体温标签 + 一句话摘要
- 战略信号仪表盘表格（信号类型 | 核心主题 | 研判结论）

**5. 晨会速览（卡片，来自备忘录视图）**
- 左列："今天必须做的事"
- 右列："今天不要做的事"

**6. 需要你做的事（卡片）**
- 来源：备忘录视图的动作队列 + 投资雷达的策略/调研建议
- 每条一行：优先级标签（P0 红色 / P1 橙色 / P2 灰色）+ 动作描述 + 右侧标的/范围

**7. 持仓一览（卡片）**
- 每只持仓一行：
  - 左侧：标的名称 + Ticker + 阶段（PVE/PVP + 具体阶段）
  - 右侧：最新动态/状态摘要

**8. 页脚**
- `YMOS Dashboard · 数据截至 YYYY-MM-DD HH:MM`

### 样式要求

- **页面宽度**：内容区最大宽度 `1200px`，居中，左右 padding `24px`（充分利用屏幕空间）
- **基础字号**：body `16px`，卡片正文 `15px`，表格内容 `14px`，标题按层级递增
- **行高**：正文 `1.7`，表格 `1.5`（确保中文阅读舒适）
- 背景色：`#f5f5f0`（暖灰）
- 卡片：白色背景、圆角 12px、轻投影、内边距 `24px`
- 字体：系统字体栈（`-apple-system, "PingFang SC", "Microsoft YaHei", sans-serif`）
- 表格：宽度 `100%`，单元格 padding `10px 12px`
- 移动端适配：工作流步骤在窄屏时 2×2 排列
- 中文为主，Ticker/价格保留英文

### 降级规则

- 当日某份报告不存在时，对应区块显示"暂无数据"，不留空
- 所有数据从已读取的文件中提取，不编造
