# P2: 个股雷达清单 (Radar)



## 路径上下文（YMOS）
- 根目录：`YMOS/`
- 主数据目录：`data/stocks/`
- 读取：`动态Watchlist/{ticker}/个股基础知识库.md` 或 `持仓/{ticker}/个股基础知识库.md`
- 可选读取：`当前关注方向与投资偏好.md`、`买入卖出备忘录.md`
- 写回：标的 `个股基础知识库.md` 增量段

## 适用场景

在完成 P1-Initiation（基石档案）后，针对已覆盖个股生成**持续跟踪的信息雷达**。目的是：
1. 防止遗漏影响投资判断的关键信号
2. 在信号出现时，快速判断其对核心投资逻辑的影响方向和幅度
3. 将日常信息流转化为结构化的决策输入

## 与 P1-Initiation 的关系

P1 建立的是静态的基石档案（公司是什么），P2 建立的是动态的监控网络（公司在发生什么变化）。P2 的每一项监控指标都应该回溯到 P1 中的核心判断——如果一个信号不会影响 P1 中任何一条关键结论，那它就不值得跟踪。

# Task

针对标的 **[{{ticker}}]**，基于其 P1 基石档案，生成一份**结构化的信息雷达清单**。

清单的核心原则：**每条监控项都必须明确标注"为什么要盯"和"怎么解读"**——不能只列"关注 XX"，而不说关注它是因为什么、出现什么变化意味着什么。

# 雷达维度

## 1. 本体核心 (The Core) — 公司自身的关键信号

### 1.1 必跟踪的官方披露

| 文件类型 | 时间节点 | 重点关注内容 |
|---------|---------|-------------|
| 10-K（年报） | 财年结束后 60 天内 | 风险因素变化、业务描述更新、客户集中度、管理层讨论（MD&A） |
| 10-Q（季报） | 季度结束后 40 天内 | 收入拆分、利润率变化、资产负债表异常、现金流质量 |
| 8-K（重大事件） | 实时 | 高管变动、重大合同、并购、诉讼、Guidance 调整 |
| Earnings Release + Call | 每季度 | 实际 vs. Guidance 偏差、管理层措辞变化、Q&A 中的新信息 |
| Proxy Statement (DEF 14A) | 年度股东大会前 | 管理层薪酬结构变化、激励指标是否与股东利益一致 |
| 月度/周度运营数据 | 如适用 | [根据公司填写：如交付量、MAU、net deposits、bookings 等] |

### 1.2 管理层行为信号

| 信号 | 解读视角 |
|------|---------|
| 高管/董事集中买入 | 通常为正面信号，但需确认是否为预设的 10b5-1 计划 |
| 高管/创始人大量卖出 | 不一定是负面（可能是流动性需求），但持续大量卖出值得警惕 |
| CFO 变更 | 高度警惕——CFO 是最了解财务真相的人，非正常离职是强烈的负面信号 |
| Guidance 措辞从具体变模糊 | 例如从"预计增长 20–25%"变为"预计继续增长"——通常预示放缓 |
| 回购节奏变化 | 加速回购可能表示管理层认为股价被低估；停止回购可能表示现金流紧张或有其他用途 |
| 股权激励指标调整 | 从 EPS 目标改为 revenue 目标？可能暗示利润增长承压 |

# 输出格式

## 雷达清单总览

最终输出应按照以下优先级排序：

### Tier 1：必须实时响应的信号（出现后 24 小时内完成解读）
- 公司财报 / Guidance 变更
- Thesis Killer 级别事件
- 核心竞品财报中直接涉及份额/定价的信号
- 重大监管/政策变化

> **响应方式**：触发 P4-Event Impact 进行深度解读。SOP 1 标的侧重评估长期逻辑是否受影响；SOP 2 标的侧重评估催化剂是否兑现/失效。

### Tier 2：需要在一周内纳入分析的信号
- 北极星指标的月度/周度更新
- 上下游产业链关键数据更新
- 管理层行为异常（大量增减持、高管变更）
- 叙事强化/弱化事件

> **响应方式**：纳入 P3-Update 定期报告。SOP 2 标的对 Tier 2 中的催化剂/情绪信号应给予更高权重。

### Tier 3：定期复盘即可（月度/季度）
- 竞品技术路线演进
- 宏观环境渐变（利率趋势、汇率趋势）
- 行业格局变化

> **响应方式**：纳入 P3-Update 月度/季度报告。SOP 1 标的对 Tier 3 中的结构性变化（行业格局、技术替代）应给予更高权重。
