# 入职引导 SOP

> 暗号：`开始使用` / `初始化系统`
> 路径基准：`YMOS/`

---

## 一句话定位

入职引导是系统的「冷启动」流程：**了解用户关注什么、持有什么、怎么想的**，初始化整个系统。

- 只在首次使用或系统状态为空时触发
- 完成后用户可以直接进入日常流程（`跑一下市场洞察`）
- 核心使命：**引导用户写好 `当前关注方向与投资偏好.md`**

---

## 触发方式

### 方式一：自动检测（每次新会话）

Agent 在会话开始时执行以下检测：

1. `data/当前关注方向与投资偏好.md` — 是否有实质内容（不只是模板）
2. `data/state/holdings.md` — 是否有持仓行
3. `data/state/watchlist.md` — 是否有关注行

**判定规则**：
- 三者全空 → 触发 **完整入职引导**
- 投资偏好为空 → 触发 **偏好补全引导**（最高优先）
- 仅缺持仓或 Watchlist → 提示补全，不强制

### 方式二：用户主动触发

| 暗号 | 操作 |
|:---|:---|
| `开始使用` | 完整入职引导 |
| `初始化系统` | 完整入职引导 |
| `补全信息` | 补全引导（只补缺失部分） |

## 完整入职引导步骤

### Step 0：Futu OpenD 检测（可选增强）

尝试连接 Futu OpenD，判断是否可自动导入数据：

```bash
uv run ymos position fetch
```

| 连接状态 | 后续行为 |
|:---|:---|
| **在线** | 提示用户：「检测到 Futu OpenD 在线，可以自动导入你的真实持仓和交易记录。是否自动导入？」→ 若用户同意，Step 2 自动填充 |
| **离线** | 跳过，不提示。入职引导正常走手动录入流程 |

> Futu OpenD 是富途证券的本地网关，支持港股/A股/美股。安装方式见下方「Futu OpenD 安装」section。

### Step 1：投资偏好深度访谈（最重要）

> 详细访谈流程见 `sop/step1-interview.md`

通过深度访谈获取用户投资偏好，写入 `data/state/preferences.md`。

### Step 2：当前持仓

**2A. Futu 自动导入**（Step 0 检测在线且用户同意时）：
```bash
uv run ymos position fetch --output-dir data/position
```
- 解析返回的持仓数据（ticker、名称、数量、成本、市值）
- 对每个标的：调用 `ymos-target-mgmt` 的「建仓」流程自动建档
- 展示导入结果供用户确认：
  ```
  从 Futu 导入了 N 个持仓：
  | Ticker | 名称 | 数量 | 市值 |
  | AAPL | 苹果 | 100 | $18,500 |
  | 0700.HK | 腾讯 | 200 | HK$78,000 |
  ...
  确认无误？我帮你初始化持仓状态机和个股文件夹。
  ```

**2B. 手动录入**（Futu 离线或用户选择手动）：
```
你目前持有哪些标的？
请告诉我 Ticker 或名称，逗号分隔即可。
例如：NIO, 腾讯, NVDA, BTC

如果当前空仓，直接说「空仓」即可。
```

对每个持仓标的（无论自动还是手动）：
1. 确认 Ticker、名称、所属市场
2. 调用 `ymos-target-mgmt` 的「建仓」流程
3. 询问是否立即跑初始调研（P1+P4+P2）
   - 标的 ≤3 个：建议同步跑，直接说 `调研一下 [TICKER]`
   - 标的 >3 个：先做基础建档，深度调研后续按需 `调研一下 [TICKER]`

### Step 3：关注清单

```
除了已持有的，你还在观察哪些标的？（尚未买入但在关注的）
同样 Ticker 或名称，逗号分隔。没有的话说「暂无」。
```

对每个关注标的：
- 调用 `ymos-target-mgmt` 的「关注」流程
- 同理，询问是否立即跑初始调研（说 `调研一下 [TICKER]` 即可触发）

### Step 4：确认与引导下一步

```
✅ YMOS 初始化完成！

📊 你的投资工作台已就绪：
- 投资偏好：已配置（9 维度）
- 持仓：X 个标的（Futu 自动导入 / 手动录入）
- 关注：Y 个标的
- Futu OpenD：在线 ✅ / 未连接（部分能力不可用）

💡 下一步你可以：
1. 说「跑一下市场洞察」— 看看今天市场发生了什么
2. 说「跑一下投资雷达」— 看看持仓和关注的价格动态
3. 说「调研一下 [TICKER]」— 深入研究某个标的
4. 说「收口一下」— 生成持仓视图（Futu 在线时含真实持仓校验）
```

## Futu OpenD 安装与配置

### 什么是 Futu OpenD

Futu OpenD 是富途证券提供的本地开放网关程序。YMOS 通过它获取：
- **真实持仓**（`ymos position fetch`）— 自动导入，省去手动录入
- **交易记录**（`ymos trade-history fetch`）— 供 P12 纪律审查和行为分析
- **资金异动**（`ymos fetch-capital-flow`）— 投资雷达的辅助信号
- **个股情绪**（`ymos fetch-sentiment`）— 评论情绪分析
- **选股筛选**（`ymos screen`）— 按条件筛选港股/A股/美股
- **技术异常**（`ymos fetch-technical-anomaly`）— 技术面异动信号
- **衍生品异动**（`ymos fetch-derivatives-anomaly`）— 期权/期货异动

### 安装步骤

1. **开立富途证券账户**（如已有则跳过）
   - 访问 [futunn.com](https://www.futunn.com/) 开户
   - 支持港股、A股（沪港通/深港通）、美股

2. **下载 Futu OpenD**
   - GitHub: [FutunnOpen/py-futu-api/releases](https://github.com/FutunnOpen/py-futu-api/releases)
   - macOS / Windows / Linux 均支持
   - 解压到任意目录

3. **启动 Futu OpenD**
   ```bash
   # macOS/Linux
   ./FutuOpenD_版本号/FutuOpenD

   # Windows
   FutuOpenD_版本号\FutuOpenD.exe
   ```

4. **登录**
   - 启动后自动打开浏览器，扫码登录富途账户
   - 默认监听 `127.0.0.1:11111`

5. **验证连接**
   ```bash
   uv run ymos position fetch
   ```
   - 成功返回持仓数据 → 配置完成
   - 连接失败 → 检查 Futu OpenD 是否已启动并登录

### 注意事项
- Futu OpenD 需保持运行状态（后台进程即可）
- YMOS 所有 Futu 相关命令在 OpenD 离线时自动跳过，**不影响核心功能**
- 不使用富途证券的用户可跳过此步骤，YMOS 仍可完整运行

## 补全引导模式

当系统检测到部分数据已填时，只补缺失的部分：

| 缺失项 | 补全动作 |
|:---|:---|
| `当前关注方向与投资偏好.md` 为空/过于简略 | **最高优先**，执行 Step 1 |
| 持仓状态机为空 | 执行 Step 2 |
| Watchlist 为空 | 执行 Step 3 |

补全模式的消息更简洁：
```
检测到你的 [缺失项] 尚未配置。我来帮你快速补全。
```

### 偏好文件「简略」判定标准

Agent 检查 `当前关注方向与投资偏好.md` 时，以下情况视为"过于简略，需要补全"：
- 缺少「投资者画像」section
- 缺少「仓位配置框架」section
- 缺少「核心心法」section
- 「板块关联逻辑」表格为空

提示用户补全时的引导语：
```
你的「当前关注方向与投资偏好」文件还比较薄。
这个文件是系统的灵魂——填得越详细，投资雷达越精准，策略建议越对味。
我来帮你逐项补充几个关键维度，大约需要 5-10 分钟。
```

---

## 边界

- 入职引导**不做市场分析**（那是市场洞察的事）
- 入职引导**不做策略推荐**（那是策略分析的事）
- 只负责：了解用户 → 初始化状态 → 写入偏好文件 → 引导下一步
- 持仓/关注的建档复用 `ymos-target-mgmt`，不重复实现
- Futu OpenD 离线时不阻塞，自动降级为手动录入
