# P8: 宏观敏感度压力测试 (Macro Filter)



## 路径上下文（YMOS）
- 根目录：`YMOS/`
- 主数据目录：`data/stocks/`
- 读取：`动态Watchlist/{ticker}/个股基础知识库.md` 或 `持仓/{ticker}/个股基础知识库.md`
- 可选读取：`当前关注方向与投资偏好.md`、`买入卖出备忘录.md`
- 写回：标的 `个股基础知识库.md` 增量段

## 适用场景

覆巢之下无完卵。当宏观环境（利率/汇率）剧变时，快速扫描哪些持仓最危险。

## 提示词模板

```
# Role
你是一名 **宏观策略分析师**。
请基于当前的 **[宏观事件]** (如：美联储加息、地缘战争、汇率波动)，对我关注的 **[{{ticker}}]** 或 **[整个持仓]** 进行压力测试。

# Context (User Input)
1. **Macro Event:** {{macro_event}}
2. **Target Ticker/Portfolio:** {{ticker}}
3. **Knowledge Base:** {{context}}

# Stress Test Framework

## 1. 贴现率敏感度 (Valuation Risk)
* **原理:** 高增长/不盈利的科技股对利率最敏感。
* **判定:** 该事件是否会导致 [Ticker] 的估值模型分母端（WACC）剧烈上升？

## 2. 业务传导机制 (Operational Risk)
* **成本端:** 原材料/能源价格上涨是否直接冲击毛利？
* **需求端:** 宏观经济放缓是否导致其客户（如企业IT支出）削减预算？

## 3. 避险属性 (Safe Haven Check)
* 在此宏观背景下，该标的是资金的"避风港" (如 BTC 在法币贬值时)，还是"提款机"？

# Output Format

## 初始评估
* **宏观冲击评级 (初始):** [高危 / 中性 / 受益]
* **逻辑传导链:** [事件] -> [影响因子] -> [公司财务影响]
* **操作建议:** [对冲保护 / 趁恐慌吸筹 / 削减杠杆]

## 辩论模式
> 完成初始评估后，立即执行辩论模式（详见下方"辩论模式"章节）。
> 输出"辩结论"三学派投票、最强反论、裁决和最终评级。
> **最终评级取代初始评级**，作为后续策略分析的正式输入。
```

## 输入参数

| 参数 | 说明 | 必填 |
|-----|------|-----|
| `macro_event` | 宏观事件描述（如：10年期美债收益率突破4.5%） | 是 |
| `ticker` | 股票代码或持仓列表 | 是 |
| `context` | 公司业务简述或基石档案 | 否 |

## 压力测试三维度

| 维度 | 检查要点 | 高危信号 |
|-----|---------|---------|
| 贴现率敏感度 | WACC 变化对估值影响 | 高增长/不盈利股 |
| 业务传导 | 成本端/需求端冲击 | 毛利下滑、客户削减预算 |
| 避险属性 | 资金流向 | 成为"提款机" |

---

## 辩论模式（每次 P8 必跑）

> **目的：** 用三大经济学派的视角对上述压力测试结论进行"钢铁人反驳"（Steelman），防止单一思维框架导致盲区。辩论结果直接影响最终宏观冲击评级。

### 三学派视角分析

针对上述宏观事件和标的，请分别从以下三个经济学派的理论框架进行分析（每派 2-3 句话）：

**1. 凯恩斯主义视角（需求侧）**
- 关注：总需求、财政刺激、消费信心、乘数效应
- 核心问题：该宏观事件对需求端的冲击有多大？政府是否有能力通过财政/货币政策对冲？
- 分析模板：该事件 → 对 {{ticker}} 所在行业的总需求影响 → 政策缓冲空间 → 结论

**2. 奥地利学派视角（供给侧/自由市场）**
- 关注：资本错配、信贷扩张、市场自我纠错、创造性破坏
- 核心问题：该宏观事件是否暴露了之前被低利率掩盖的结构性问题？市场纠错对 {{ticker}} 意味着什么？
- 分析模板：该事件 → 揭示的结构性扭曲程度 → 市场纠错路径 → 对 {{ticker}} 的影响 → 结论

**3. 重商主义视角（国家战略/贸易）**
- 关注：产业政策、贸易壁垒、国家竞争力、供应链安全
- 核心问题：该宏观事件是否改变了 {{ticker}} 所在行业的全球竞争格局？国家政策会保护还是放弃该行业？
- 分析模板：该事件 → 对全球产业格局的重塑 → 国家政策倾向 → {{ticker}} 的战略价值变化 → 结论

### Steelman 反驳机制

在三学派分析完成后，执行以下反驳程序：

1. **寻找最强反论：** 从三学派中识别出对"宏观冲击评级: [高危 / 中性 / 受益]"这一原始结论最强有力的反驳。这个反驳必须是理性的、有数据或逻辑支撑的，不能是稻草人论证。
2. **评估反驳强度：** 判断该反驳是否足以推翻或修改原始评级（强反驳/中等反驳/弱反驳）。
3. **决策：** 说明是否采纳该反驳，以及采纳或拒绝的理由。

### 辩结论输出格式

```
## 辩结论

### 三学派投票
* **凯恩斯主义:** [高危 / 中性 / 受益] — [一句话理由]
* **奥地利学派:** [高危 / 中性 / 受益] — [一句话理由]
* **重商主义:** [高危 / 中性 / 受益] — [一句话理由]

### 最强反论
* **来源学派:** [哪个学派提出了最强反驳]
* **反论内容:** [2-3 句话阐述该反论的核心逻辑]
* **反驳强度:** [强反驳 / 中等反驳 / 弱反驳]

### 裁决
* **是否采纳:** [采纳 / 拒绝]
* **理由:** [为什么采纳或拒绝，1-2 句话]
* **最终评级:** [高危 / 中性 / 受益]（可能与原始评级相同或不同）
* **评级调整说明:** [若评级变化，说明变化原因；若未变化，说明为何原评级经受住了反驳]
```

> **规则：** 辩结论中的"最终评级"取代原始的"宏观冲击评级"，作为后续策略分析的正式输入。策略报告必须引用辩结论，而非仅引用原始评级。
